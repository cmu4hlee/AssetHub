# AssetHub 模块化架构实施报告

## 一、实施概述

按照《医学装备整体运维管理服务规范》的要求，将新功能以独立模块的形式实现，支持灵活配置和按需启用。

---

## 三、创建的模块

### 模块 1: 合规性管理 (compliance-management)

**模块ID**: `compliance-management`

**功能特性**:
| 功能ID | 名称 | 说明 | 可配置 |
|--------|------|------|--------|
| `maintenance_level` | 分级保养 | 日常/一级/二级/三级保养体系 | ✅ |
| `risk_assessment` | 风险评估 | 设备风险等级评估与管理 | ✅ |
| `special_equipment` | 特种设备 | 特种设备台账与检验管理 | ✅ |
| `safety_inspection` | 安全检测 | 电气/辐射等安全检测管理 | ✅ |

**API端点**: `/api/compliance/*`

**配置项**:
- `maintenance_level_enabled` - 启用分级保养 (默认: true)
- `risk_assessment_enabled` - 启用风险评估 (默认: true)
- `special_equipment_enabled` - 启用特种设备管理 (默认: true)
- `safety_inspection_enabled` - 启用安全检测 (默认: true)
- `default_inspection_reminder_days` - 检验提醒提前天数 (默认: 90)
- `enable_auto_plan_generation` - 自动生成保养计划 (默认: true)

---

### 模块 2: 开机率管理 (uptime-management)

**模块ID**: `uptime-management`

**功能特性**:
| 功能ID | 名称 | 说明 | 可配置 |
|--------|------|------|--------|
| `operation_log` | 运行记录 | 设备每日运行时间记录 | ✅ |
| `uptime_statistics` | 开机率统计 | 按月统计设备开机率 | ✅ |
| `uptime_analysis` | 开机率分析 | 开机率趋势分析与预警 | ✅ |
| `downtime_analysis` | 停机分析 | 停机原因分类与分析 | ✅ |

**API端点**: `/api/uptime/*`

**配置项**:
- `operation_log_enabled` - 启用运行记录 (默认: true)
- `auto_calculation_enabled` - 自动计算开机率 (默认: true)
- `life_support_uptime_threshold` - 生命支持类设备开机率阈值 (默认: 99%)
- `large_equipment_uptime_threshold` - 大型设备开机率阈值 (默认: 95%)
- `regular_equipment_uptime_threshold` - 常规设备开机率阈值 (默认: 95%)
- `low_uptime_warning_enabled` - 低开机率预警 (默认: true)
- `data_collection_method` - 数据采集方式 (默认: manual)

---

### 模块 3: 人员资质管理 (staff-qualification)

**模块ID**: `staff-qualification`

**功能特性**:
| 功能ID | 名称 | 说明 | 可配置 |
|--------|------|------|--------|
| `qualification_management` | 资质管理 | 专业资质、技能证书管理 | ✅ |
| `training_management` | 培训管理 | 培训计划、培训记录管理 | ✅ |
| `qualification_reminder` | 资质到期提醒 | 资质到期前自动提醒 | ✅ |
| `competency_assessment` | 能力考核 | 人员技能考核与评估 | ✅ |

**API端点**: `/api/staff/*`

**配置项**:
- `qualification_management_enabled` - 启用资质管理 (默认: true)
- `training_management_enabled` - 启用培训管理 (默认: true)
- `qualification_reminder_days` - 资质到期提醒天数 (默认: 90)
- `training_reminder_days` - 培训到期提醒天数 (默认: 30)
- `mandatory_training_enabled` - 强制培训要求 (默认: true)
- `qualification_types` - 资质类型 (默认: [professional, skill, safety, special])

### 模块 4: 设备风险管理 (asset-risk-management)

**模块ID**: `asset-risk-management`

**功能特性**:
| 功能ID | 名称 | 说明 | 可配置 |
|--------|------|------|--------|
| `risk_assessment` | 风险评估 | 设备风险评估与评分 | ✅ |
| `risk_classification` | 风险分级 | 设备风险等级分类管理 | ✅ |
| `risk_control` | 风险控制 | 风险控制措施制定与跟踪 | ✅ |
| `risk_monitoring` | 风险监控 | 风险指标监控与预警 | ✅ |

**API端点**: `/api/risk/*`

**配置项**:
- `risk_assessment_enabled` - 启用风险评估 (默认: true)
- `auto_risk_classification` - 自动风险分级 (默认: true)
- `high_risk_threshold` - 高风险阈值 (默认: 80)
- `medium_risk_threshold` - 中风险阈值 (默认: 50)
- `risk_assessment_cycle_months` - 风险评估周期 (默认: 12月)
- `high_risk_alert_enabled` - 高风险设备预警 (默认: true)
- `risk_assessment_items` - 风险评估项目 (默认: [importance, usage_frequency, failure_history, age, safety_risk])

## 四、模块管理API

### 获取所有模块状态
```http
GET /api/system/modules
```

响应示例:
```json
{
  "success": true,
  "data": [
    {
      "id": "compliance-management",
      "name": "合规性管理",
      "version": "1.0.0",
      "enabled": true,
      "features": [
        { "id": "maintenance_level", "name": "分级保养", "enabled": true },
        { "id": "special_equipment", "name": "特种设备", "enabled": true }
      ]
    }
  ]
}
```

### 获取模块详情
```http
GET /api/system/modules/:id
```

### 获取模块配置
```http
GET /api/{module-prefix}/config
```

示例:
```http
GET /api/uptime/config
```

响应:
```json
{
  "success": true,
  "data": {
    "schema": [...],
    "default": { ... }
  }
}
```

### 获取模块状态
```http
GET /api/{module-prefix}/status
```

示例:
```http
GET /api/compliance/status
```

响应:
```json
{
  "success": true,
  "data": {
    "module_id": "compliance-management",
    "name": "合规性管理",
    "version": "1.0.0",
    "features": [...],
    "config": { ... }
  }
}
```

## 六、配置灵活性

### 6.1 多租户支持

每个租户可以独立配置模块:
- 启用/禁用整个模块
- 启用/禁用模块内特定功能
- 自定义模块配置参数

### 6.2 热更新能力

- 无需重启服务即可切换功能开关
- 模块配置支持实时更新
- 支持模块动态加载

### 6.3 依赖管理

模块间依赖自动检查:
```javascript
dependencies: [
  { module_id: 'asset-management', dependency_type: 'required' },
  { module_id: 'maintenance-management', dependency_type: 'optional' }
]
```
