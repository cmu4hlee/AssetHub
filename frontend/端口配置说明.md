# 前端端口配置说明

## 📌 端口配置

**开发模式：前端6000，后端6001**
**生产模式：前端4000，后端4001**

### 配置位置

1. **vite.config.js**
   - 开发环境（`server.port`）: 默认 6000
   - 生产环境（`preview.port`）: 默认 4000
   - 可通过环境变量 `VITE_FRONTEND_PORT` 和 `VITE_PREVIEW_PORT` 覆盖

2. **环境变量**
   ```bash
   # 开发环境端口（默认6000）
   VITE_FRONTEND_PORT=6000
   
   # 生产预览端口（默认4000）
   VITE_PREVIEW_PORT=4000
   ```

## ⚠️ 重要提示

### 端口说明

**开发模式**：
- 前端默认端口：6000
- 后端默认端口：6001
- 如果6000端口被占用，系统会自动尝试下一个可用端口（6002、6003等）

**生产模式**：
- 前端默认端口：4000
- 后端默认端口：4001
- 如果4000端口被占用，系统会自动尝试下一个可用端口（4001、4002等）

如果需要使用其他端口，可以通过环境变量配置：

```bash
# 开发环境使用其他端口
VITE_FRONTEND_PORT=6002 npm run dev

# 生产环境使用其他端口
VITE_PREVIEW_PORT=4002 npm run preview
```

#### Windows
Windows 系统通常不需要特殊权限，可以直接使用80端口。

### 如果80端口被占用

如果80端口已被其他服务占用，系统会**自动尝试下一个可用端口**（81、82、83...），无需手动配置。

**自动端口选择机制**：
- 开发环境：80 -> 81 -> 82 -> ...
- 生产环境：80 -> 81 -> 82 -> ...

**查看实际使用的端口**：
- 查看服务启动窗口的输出信息
- 查看浏览器地址栏显示的端口号

**手动指定端口**（如果需要）：
```bash
# 使用环境变量覆盖
VITE_FRONTEND_PORT=8080 npm run dev
VITE_PREVIEW_PORT=8080 npm run preview
```

**检查端口占用**：
```bash
# macOS/Linux
lsof -i :4000

# Windows
netstat -ano | findstr :4000
```

## 🚀 启动命令

### 开发环境
```bash
# 默认6000端口（开发模式）
npm run dev

# 或指定端口
VITE_FRONTEND_PORT=6002 npm run dev
```

### 生产环境预览
```bash
# 默认4000端口（生产模式）
npm run preview

# 或指定端口
VITE_PREVIEW_PORT=4002 npm run preview
```

## 📝 注意事项

1. **端口区分**：
   - 开发模式：前端6000，后端6001
   - 生产模式：前端4000，后端4001
2. **无需权限**：这些端口都是普通端口，不需要管理员权限
3. **端口冲突**：如果端口被占用，系统会自动尝试下一个可用端口
4. **代理配置**：后端代理配置会自动适配，无需修改
