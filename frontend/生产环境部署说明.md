# 生产环境部署说明

## 前端端口配置

### 开发环境
- **端口**: 4000（开发和生产环境统一）
- **启动命令**: `npm run dev`
- **访问地址**: http://localhost:4000

### 生产环境
- **端口**: 4000（如果被占用，自动尝试4001、4002等端口）
- **启动命令**: 
  1. 先构建: `npm run build`
  2. 再预览: `npm run preview` 或 `npm run preview:prod`
- **访问地址**: http://localhost:4000 (或 http://your-domain:4000)
- **自动端口选择**: 如果4000端口被占用，系统会自动尝试下一个可用端口

## 注意事项

### 1. 端口权限
4000 端口是普通端口，不需要 root 权限，可以直接使用。

如果遇到端口被占用的问题，可以：

**方案1：使用环境变量指定其他端口**
```bash
VITE_PREVIEW_PORT=4001 npm run preview
```

**方案2：使用端口转发（如果需要使用80端口）**
```bash
# 使用非特权端口（如 8080），然后通过 iptables 转发到 80
sudo iptables -t nat -A PREROUTING -p tcp --dport 80 -j REDIRECT --to-port 8080
npm run preview -- --port 8080
```

**方案3：使用 Nginx 反向代理（生产环境推荐）**
```nginx
server {
    listen 80;
    server_name your-domain.com;
    
    root /path/to/frontend/dist;
    index index.html;
    
    location / {
        try_files $uri $uri/ /index.html;
    }
    
    location /api {
        proxy_pass http://localhost:4001;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
    
    location /uploads {
        proxy_pass http://localhost:4001;
    }
}
```

### 2. 防火墙配置
确保防火墙允许 80 端口访问：
```bash
# Ubuntu/Debian
sudo ufw allow 80/tcp

# CentOS/RHEL
sudo firewall-cmd --permanent --add-port=80/tcp
sudo firewall-cmd --reload
```

### 3. 后端代理配置
生产环境的前端预览服务器会自动代理 `/api` 和 `/uploads` 到后端（`http://localhost:4001`）。

如果后端运行在不同的服务器或端口，需要修改 `vite.config.js` 中的 `preview.proxy` 配置。

## 部署步骤

### 方式1：使用 Vite Preview（简单）
```bash
cd Asset/frontend
npm run build
npm run preview
```

### 方式2：使用 Nginx（推荐生产环境）
```bash
# 1. 构建前端
cd Asset/frontend
npm run build

# 2. 配置 Nginx（见上面的配置示例）

# 3. 启动 Nginx
sudo systemctl start nginx
```

### 方式3：使用 PM2（Node.js 进程管理）
```bash
# 1. 构建前端
cd Asset/frontend
npm run build

# 2. 安装 serve（静态文件服务器）
npm install -g serve

# 3. 使用 PM2 启动
pm2 serve dist 80 --name "frontend" --spa
```

## 验证部署

1. **检查端口是否监听**:
   ```bash
   # Linux/macOS
   lsof -i :80
   # 或
   netstat -tuln | grep 80
   
   # Windows
   netstat -ano | findstr :80
   ```

2. **测试访问**:
   ```bash
   curl http://localhost
   ```

3. **检查后端代理**:
   ```bash
   curl http://localhost/api/health
   ```

## 常见问题

### Q: 端口 80 被占用
**A**: 检查并停止占用 80 端口的服务，或使用 Nginx 反向代理。

### Q: 权限不足
**A**: 使用 sudo 运行（不推荐）或使用 Nginx 等 web 服务器。

### Q: 代理不工作
**A**: 检查后端服务是否运行在 4001 端口，或修改 `vite.config.js` 中的代理配置。

### Q: 构建后的文件在哪里？
**A**: 构建后的文件在 `dist` 目录中。

## 安全建议

1. **不要使用 root 权限运行 Node.js 应用**
2. **使用 Nginx 等专业 web 服务器**处理静态文件和反向代理
3. **配置 HTTPS**（使用 443 端口）
4. **设置适当的文件权限**
5. **定期更新依赖包**
