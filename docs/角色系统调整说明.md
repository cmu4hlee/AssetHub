# 角色系统调整说明

## 一、角色调整概述

### 调整前
- **system_admin（系统管理员）**：可以查看和管理所有租户的数据

### 调整后
- **system_admin（系统管理员）**：每个租户的系统管理员，只能管理租户范围内的功能
- **super_admin（超级管理员）**：整个系统的管理员，可以管理所有租户和系统功能

## 二、角色权限对比

| 功能 | super_admin | system_admin | 普通用户 |
|------|-------------|--------------|----------|
| 查看所有租户数据 | ✅ | ❌ | ❌ |
| 查看自己租户数据 | ✅ | ✅ | ✅ |
| 企业管理（租户管理） | ✅ | ❌ | ❌ |
| 用户管理 | ✅ | ✅（租户内） | ❌ |
| 角色权限管理 | ✅ | ✅（租户内） | ❌ |
| 操作日志 | ✅（所有） | ✅（租户内） | ✅（自己） |
| 资产管理 | ✅（所有） | ✅（租户内） | ✅（科室） |
| 其他业务功能 | ✅（所有） | ✅（租户内） | ✅（科室） |

## 三、技术实现

### 1. 租户过滤逻辑

**文件**: `backend/middleware/tenant-filter.js`

```javascript
function addTenantFilter(req, tableAlias = '') {
  // 超级管理员不添加租户过滤，可以查看所有数据
  // 系统管理员（租户级）和普通用户添加租户过滤
  if (req.user.role !== 'super_admin' && req.user.tenant_id) {
    whereClause = ` AND ${prefix}tenant_id = ?`;
    params.push(req.user.tenant_id);
  }
}
```

### 2. 认证中间件

**文件**: `backend/middleware/auth.js`

- 超级管理员不检查租户状态
- 超级管理员可以访问所有科室
- 系统管理员（租户级）只能访问租户内的科室

### 3. 权限检查

- `requireSuperAdmin`: 仅超级管理员可访问（如企业管理）
- `requireSystemAdmin`: 超级管理员和系统管理员（租户级）都可访问
- `authorize`: 超级管理员和系统管理员（租户级）都拥有所有权限

### 4. 登录逻辑

**文件**: `backend/routes/users.js`

- 超级管理员：不需要企业编码，不关联租户
- 系统管理员（租户级）：需要租户，登录时可以不提供企业编码（使用已有租户）
- 普通用户：必须提供企业编码

## 四、超级管理员账户

### 创建方式

运行脚本：
```bash
cd Asset/backend
node scripts/create-super-admin.js
```

### 账户信息

- **用户名**: `suadmin`
- **密码**: `123`
- **角色**: `super_admin`
- **租户**: 无（不关联租户）

## 五、前端调整

### 1. 登录页面

- 支持超级管理员和系统管理员（租户级）登录
- 勾选"系统管理员/超级管理员登录"时，不需要输入企业编码

### 2. 菜单显示

- 超级管理员可以看到"企业管理"菜单
- 系统管理员（租户级）看不到"企业管理"菜单
- 其他系统管理菜单两者都可以看到

### 3. 菜单可见性

- 超级管理员和系统管理员（租户级）默认显示所有菜单
- 普通用户根据权限显示菜单

## 六、数据库变更

### 用户表（users）

- `role` 字段支持 `super_admin` 值
- 超级管理员的 `tenant_id` 为 `NULL`

### 角色表（roles）

如果需要，可以添加 `super_admin` 角色定义。

## 七、注意事项

1. **数据隔离**：
   - 超级管理员可以查看所有租户的数据
   - 系统管理员（租户级）只能查看自己租户的数据
   - 普通用户只能查看自己科室的数据

2. **企业管理**：
   - 只有超级管理员可以创建、修改、删除租户
   - 系统管理员（租户级）不能管理租户

3. **用户管理**：
   - 超级管理员可以管理所有租户的用户
   - 系统管理员（租户级）只能管理自己租户的用户

4. **操作日志**：
   - 超级管理员可以查看所有租户的操作日志
   - 系统管理员（租户级）只能查看自己租户的操作日志
   - 普通用户只能查看自己的操作日志

## 八、迁移说明

### 现有系统管理员处理

如果现有系统管理员需要保留跨租户权限：
1. 将其角色改为 `super_admin`
2. 将其 `tenant_id` 设置为 `NULL`

如果现有系统管理员需要限制为租户级：
1. 保持角色为 `system_admin`
2. 确保其 `tenant_id` 不为空

### 脚本执行

```bash
# 创建超级管理员
cd Asset/backend
node scripts/create-super-admin.js
```
