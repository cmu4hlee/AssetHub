# 外部访问前端配置说明

## 🔍 问题诊断

当通过 `http://192.168.1.80:6000` 访问前端失败时，可能的原因：

### 1. 服务未启动
**检查方法**：
```bash
lsof -i:6000
# 或
ps aux | grep vite
```

**解决方法**：
```bash
cd Asset/frontend
npm run dev
```

### 2. 防火墙阻止
**检查方法**：
```bash
# macOS 检查防火墙状态
sudo /usr/libexec/ApplicationFirewall/socketfilterfw --getglobalstate

# 检查端口是否被阻止
sudo /usr/libexec/ApplicationFirewall/socketfilterfw --listapps
```

**解决方法**：
- macOS：系统设置 > 安全性与隐私 > 防火墙 > 选项 > 允许 Node.js 接收传入连接
- 或临时关闭防火墙测试

### 3. 网络配置问题
**检查方法**：
```bash
# 检查本机IP
ifconfig | grep "inet "

# 测试本地访问
curl http://localhost:6000

# 测试外部IP访问
curl http://192.168.1.80:6000
```

### 4. Vite配置问题
**当前配置**：
- `host: '0.0.0.0'` ✅ 已配置，允许外部访问
- `port: 6000` ✅ 已配置

**验证配置**：
检查 `frontend/vite.config.js` 中的 `server.host` 是否为 `'0.0.0.0'`

## ✅ 解决方案

### 方案1：重启前端服务（推荐）

如果配置已正确，重启服务可能解决问题：

```bash
# 停止当前服务（Ctrl+C）
# 然后重新启动
cd Asset/frontend
npm run dev
```

启动后，Vite会显示：
```
VITE v7.x.x  ready in xxx ms

➜  Local:   http://localhost:6000/
➜  Network: http://192.168.1.80:6000/
```

### 方案2：检查防火墙设置

**macOS**：
1. 打开"系统设置" > "网络" > "防火墙"
2. 点击"选项"
3. 确保 Node.js 或相关应用允许传入连接

### 方案3：使用localhost访问测试

先测试本地访问是否正常：
```bash
# 在浏览器访问
http://localhost:6000
```

如果localhost可以访问，说明服务正常，问题可能是网络或防火墙。

### 方案4：检查网络连接

**在同一网络的其他设备上测试**：
```bash
# 从其他设备访问
curl http://192.168.1.80:6000
```

**检查路由器和网络设置**：
- 确保设备在同一局域网
- 检查路由器是否阻止了设备间通信

## 🔧 快速诊断命令

```bash
# 1. 检查服务是否运行
lsof -i:6000

# 2. 检查本机IP
ifconfig | grep "inet " | grep -v "127.0.0.1"

# 3. 测试本地访问
curl http://localhost:6000

# 4. 测试外部IP访问
curl http://192.168.1.80:6000

# 5. 检查防火墙
sudo /usr/libexec/ApplicationFirewall/socketfilterfw --getglobalstate
```

## 📝 配置验证

### 正确的配置应该是：

```javascript
// vite.config.js
server: {
  host: '0.0.0.0',  // 允许外部访问
  port: 6000,       // 开发模式端口
  strictPort: false,
}
```

### 启动后应该看到：

```
VITE v7.x.x  ready in xxx ms

➜  Local:   http://localhost:6000/
➜  Network: http://192.168.1.80:6000/
```

## ⚠️ 常见问题

### Q: 为什么localhost可以访问，但IP地址不行？
**A**: 可能是防火墙阻止，或网络配置问题。检查防火墙设置。

### Q: 服务显示运行，但浏览器无法访问？
**A**: 
1. 清除浏览器缓存
2. 尝试无痕模式
3. 检查浏览器控制台错误信息
4. 尝试其他浏览器

### Q: 从其他设备无法访问？
**A**:
1. 确保设备在同一局域网
2. 检查路由器设置
3. 检查macOS防火墙设置
4. 尝试关闭防火墙测试

## 🔄 重启服务

如果修改了配置，需要重启前端服务：

```bash
# 停止服务（在运行服务的终端按 Ctrl+C）
# 然后重新启动
cd Asset/frontend
npm run dev
```
