# 质量控制模块使用说明

**创建日期**: 2026-01-14  
**模块功能**: 计量管理、质控管理

---

## 📋 模块概述

质量控制模块包含两个子模块：
1. **计量管理** - 管理资产的计量检定、校准等记录
2. **质控管理** - 管理资产的质量控制检查记录

---

## 🗄️ 数据库表结构

### 1. 计量管理表 (metrology_records)

**主要字段**:
- `record_no` - 计量单号（唯一）
- `asset_id` - 资产ID
- `asset_code` - 资产编号
- `asset_name` - 资产名称
- `metrology_type` - 计量类型（首次检定、周期检定、校准、期间核查、其他）
- `metrology_date` - 计量日期
- `next_metrology_date` - 下次计量日期
- `metrology_agency` - 计量机构
- `certificate_no` - 证书编号
- `result` - 计量结果（合格、不合格、限用、待检）
- `accuracy_level` - 准确度等级
- `measurement_range` - 测量范围
- `cost` - 计量费用
- `operator` - 操作人
- `status` - 状态（待检、进行中、已完成、已取消）

### 2. 质控管理表 (quality_control_records)

**主要字段**:
- `record_no` - 质控单号（唯一）
- `asset_id` - 资产ID
- `asset_code` - 资产编号
- `asset_name` - 资产名称
- `qc_type` - 质控类型（日常质控、定期质控、专项质控、验收质控、其他）
- `qc_date` - 质控日期
- `next_qc_date` - 下次质控日期
- `qc_item` - 质控项目
- `qc_method` - 质控方法
- `qc_standard` - 质控标准
- `result` - 质控结果（合格、不合格、待检、整改中）
- `qc_value` - 质控数值
- `standard_value` - 标准值
- `deviation` - 偏差
- `operator` - 操作人
- `reviewer` - 审核人
- `review_date` - 审核日期
- `status` - 状态（待检、进行中、已完成、已取消、整改中）

## 🚀 初始化步骤

### 1. 创建数据库表

运行以下脚本创建数据库表：

```bash
cd Asset/backend
node scripts/create-quality-control-tables.js
```

### 2. 验证表创建

检查以下表是否创建成功：
- `metrology_records` - 计量管理表
- `metrology_attachments` - 计量附件表
- `quality_control_records` - 质控管理表
- `quality_control_attachments` - 质控附件表

## 🔌 API接口

### 计量管理接口

- `GET /api/quality-control/metrology` - 获取计量记录列表
- `GET /api/quality-control/metrology/:id` - 获取计量记录详情
- `POST /api/quality-control/metrology` - 创建计量记录
- `PUT /api/quality-control/metrology/:id` - 更新计量记录
- `DELETE /api/quality-control/metrology/:id` - 删除计量记录
- `POST /api/quality-control/metrology/:id/attachments` - 上传计量附件

### 质控管理接口

- `GET /api/quality-control/quality-control` - 获取质控记录列表
- `GET /api/quality-control/quality-control/:id` - 获取质控记录详情
- `POST /api/quality-control/quality-control` - 创建质控记录
- `PUT /api/quality-control/quality-control/:id` - 更新质控记录
- `DELETE /api/quality-control/quality-control/:id` - 删除质控记录
- `POST /api/quality-control/quality-control/:id/attachments` - 上传质控附件

## 🔐 权限控制

- **系统管理员**: 可以查看和管理所有科室的计量和质控记录
- **科室管理员**: 只能查看和管理自己管理科室的资产相关记录
- **普通用户**: 根据角色权限配置

---

## 📝 使用流程

### 计量管理流程

1. **创建计量记录**
   - 选择资产
   - 填写计量信息（类型、日期、机构等）
   - 保存记录

2. **更新计量结果**
   - 编辑记录
   - 更新计量结果和状态
   - 上传计量证书等附件

3. **查看计量历史**
   - 在列表页面查看所有计量记录
   - 可以按资产、类型、结果等筛选

### 质控管理流程

1. **创建质控记录**
   - 选择资产
   - 填写质控信息（类型、项目、方法、标准等）
   - 保存记录

2. **记录质控结果**
   - 填写质控数值、标准值、偏差
   - 设置质控结果和状态
   - 填写审核信息

3. **查看质控历史**
   - 在列表页面查看所有质控记录
   - 可以按资产、类型、结果等筛选

## ⚠️ 注意事项

1. **数据库初始化**: 首次使用前必须运行创建表脚本
2. **文件上传**: 附件存储在 `backend/uploads/quality-control/` 目录
3. **权限控制**: 非系统管理员只能操作自己管理科室的资产
4. **数据验证**: 计量单号和质控单号必须唯一
5. **事务处理**: 所有创建、更新、删除操作都使用数据库事务
