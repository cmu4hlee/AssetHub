# 开发模式启动问题排查指南

## 常见问题

### 1. 端口6000被占用（开发模式前端端口）

**问题现象**：
- 前端服务启动失败
- 提示端口已被占用

**原因**：
- 其他应用占用了端口 6000

**解决方案**：

#### 方案A：关闭占用端口的进程
```bash
# 查找占用端口的进程
lsof -ti:6000

# 关闭进程（替换PID为实际进程ID）
kill -9 <PID>
```

#### 方案B：使用其他端口
设置环境变量使用其他端口：
```bash
VITE_FRONTEND_PORT=6002 npm run dev
```

#### 方案C：修改默认端口
编辑 `frontend/vite.config.js`，将默认端口改为其他值（如 6002）

### 2. 端口6001被占用（开发模式后端端口）

**问题现象**：
- 后端服务启动失败
- 提示端口已被占用

**解决方案**：

#### 方案A：关闭占用端口的进程
```bash
# 查找占用端口的进程
lsof -ti:6001

# 关闭进程（替换PID为实际进程ID）
kill -9 <PID>
```

#### 方案B：使用其他端口
设置环境变量：
```bash
PORT=6002 npm start
```

### 3. 前后端端口冲突

**问题现象**：
- 前端自动切换到6001，与后端冲突

**解决方案**：
- 前端配置了 `strictPort: false`，会自动尝试下一个可用端口
- 如果6000被占用，前端会尝试6002、6003等
- 确保后端使用6001，前端使用其他端口

### 4. 数据库连接失败

**问题现象**：
- 后端启动后无法连接数据库
- 提示数据库连接错误

**解决方案**：
1. 检查 `backend/.env` 文件配置
2. 确认数据库服务正在运行
3. 检查数据库用户名和密码
4. 确认数据库端口和主机地址正确

### 5. 依赖未安装

**问题现象**：
- 启动时提示模块未找到
- npm 错误

**解决方案**：
```bash
# 安装后端依赖
cd backend
npm install

# 安装前端依赖
cd ../frontend
npm install
```

## 快速诊断命令

### 检查端口占用
```bash
# 检查前端端口（开发模式）
lsof -i:6000

# 检查后端端口（开发模式）
lsof -i:6001

# 检查所有相关端口
lsof -i:6000,6001,6002,6003
```

### 检查服务状态
```bash
# 检查Node.js进程
ps aux | grep node

# 检查Vite进程
ps aux | grep vite
```

### 检查环境变量
```bash
# 检查NODE_ENV
echo $NODE_ENV

# 检查端口配置
echo $PORT
echo $VITE_FRONTEND_PORT
```

## 标准启动流程

### 1. 确保端口可用
```bash
# 检查并关闭占用端口的进程（开发模式）
lsof -ti:6000 | xargs kill -9 2>/dev/null
lsof -ti:6001 | xargs kill -9 2>/dev/null
```

### 2. 启动后端
```bash
cd backend
npm start
# 或开发模式（自动重启）
npm run dev
```

### 3. 启动前端
```bash
cd frontend
npm run dev
```

### 4. 验证服务
- 前端：http://localhost:6000（开发模式，如果被占用会自动切换）
- 后端：http://localhost:6001/api/health（开发模式）
- 生产模式：前端4000，后端4001

## 使用启动脚本

### Windows
```bash
启动服务_windows.bat
```

### macOS/Linux
```bash
./start.sh
```

启动脚本会自动：
- 检查依赖
- 安装缺失的依赖
- 启动前后端服务
- 显示服务地址
