# 网络错误排查指南

## 错误信息
```
api.js:40 API Error: AxiosError {message: 'Network Error', name: 'AxiosError', code: 'ERR_NETWORK', ...}
```

## 可能的原因

### 1. 后端服务未运行
**症状**: 所有API请求都返回 `ERR_NETWORK` 错误

**解决方法**:
```bash
# 检查后端服务是否运行
cd Asset/backend
npm start

# 或者使用 PM2
pm2 start server.js

# 检查端口是否被占用
lsof -i :4001  # macOS/Linux
netstat -ano | findstr :4001  # Windows
```

### 2. 端口配置不匹配
**症状**: 前端代理无法连接到后端

**检查配置**:
- 前端代理配置: `frontend/vite.config.js` - 应该指向 `http://localhost:4001`
- 后端端口配置: `backend/config/app.config.js` - 默认端口是 `4001`

**解决方法**:
```javascript
// frontend/vite.config.js
proxy: {
  '/api': {
    target: 'http://localhost:4001',  // 确保与后端端口一致
    changeOrigin: true,
  }
}
```

### 3. CORS 问题
**症状**: 浏览器控制台显示 CORS 错误

**解决方法**:
检查 `backend/server.js` 中的 CORS 配置：
```javascript
app.use(cors({
  origin: '*',  // 开发环境允许所有源
  credentials: true,
}));
```

### 4. 防火墙或网络限制
**症状**: 本地可以访问，但外网无法访问

**解决方法**:
- 检查防火墙设置
- 确保后端监听 `0.0.0.0` 而不是 `127.0.0.1`
- 检查端口是否被防火墙阻止

### 5. 代理配置问题
**症状**: 开发环境无法连接，但直接访问后端可以

**解决方法**:
1. 检查 Vite 开发服务器是否正常运行
2. 检查代理配置是否正确
3. 尝试重启前端开发服务器

## 快速诊断步骤

### 步骤 1: 检查后端服务
```bash
# 在浏览器中直接访问健康检查接口
curl http://localhost:4001/api/health

# 或者
curl http://localhost:4001/api/users/login
```

如果返回错误，说明后端服务未运行或端口不对。

### 步骤 2: 检查前端代理
```bash
# 启动前端开发服务器
cd Asset/frontend
npm run dev

# 在浏览器中访问
http://localhost:4000/api/health
```

如果返回错误，说明代理配置有问题。

### 步骤 3: 检查网络连接
在浏览器开发者工具的 Network 标签中：
1. 查看请求的 URL 是否正确
2. 查看请求状态码
3. 查看响应内容

### 步骤 4: 查看详细错误信息
打开浏览器控制台，查看完整的错误信息：
- 错误代码
- 错误消息
- 请求配置
- 响应信息（如果有）

## 常见错误代码

| 错误代码 | 含义 | 解决方法 |
|---------|------|---------|
| `ERR_NETWORK` | 网络连接失败 | 检查后端服务是否运行 |
| `ECONNREFUSED` | 连接被拒绝 | 检查端口是否正确，服务是否运行 |
| `ETIMEDOUT` | 请求超时 | 检查网络连接，增加超时时间 |
| `ERR_BAD_REQUEST` | 请求错误 | 检查请求参数和格式 |
| `ERR_BAD_RESPONSE` | 响应错误 | 检查后端返回的数据格式 |

## 改进的错误处理

代码已经改进了错误处理，现在会：
1. 显示详细的错误日志
2. 区分不同类型的网络错误
3. 提供用户友好的错误提示
4. 自动检测后端服务状态

## 预防措施

1. **添加健康检查**: 在应用启动时检查后端服务是否可用
2. **添加重试机制**: 对于网络错误，自动重试
3. **添加离线检测**: 检测网络连接状态
4. **添加错误上报**: 将错误信息发送到日志系统
