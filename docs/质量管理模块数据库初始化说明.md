# 质量管理模块数据库初始化说明

**问题**: `Table 'zcgl.metrology_records' doesn't exist`

---

## 🔧 解决方法

### 方法1：通过 API 自动创建（推荐）

当访问质量管理模块的 API 时，系统会自动检测表是否存在，如果不存在会自动创建。

**操作步骤**：
1. 确保后端服务正在运行
2. 访问质量管理模块的任何页面或 API
3. 系统会自动创建所需的表

### 方法2：手动执行 SQL 脚本

如果方法1不可用，可以手动执行 SQL 脚本：

#### 步骤1：打开 MySQL 客户端

```bash
mysql -u root -p zcgl
```

#### 步骤2：执行 SQL 脚本

```bash
source /Users/cjlee/Desktop/Mac_Win_share/资产管理系统/资产管理vite/Asset/backend/scripts/create-quality-management-tables.sql
```

或者直接复制 SQL 内容到 MySQL 客户端执行。

#### 步骤3：验证表是否创建成功

```sql
SHOW TABLES LIKE '%metrology%';
SHOW TABLES LIKE '%quality%';
```

应该看到以下表：
- `metrology_records` - 计量管理表
- `metrology_attachments` - 计量附件表
- `quality_control_records` - 质控管理表
- `quality_control_attachments` - 质控附件表
- `quality_management_alerts` - 质量管理预警记录表
- `quality_management_cycles` - 质量管理周期配置表

### 方法3：使用 Node.js 脚本（需要数据库连接）

```bash
cd Asset/backend
node scripts/create-quality-control-tables.js
```

**注意**：此方法需要数据库服务正在运行且脚本能连接到数据库。

---

## 📋 需要创建的表

1. **metrology_records** - 计量管理表
2. **metrology_attachments** - 计量附件表

4. **quality_control_attachments** - 质控附件表
5. **quality_management_alerts** - 质量管理预警记录表（新增）
6. **quality_management_cycles** - 质量管理周期配置表（新增）

## ✅ 验证

表创建成功后，再次访问质量管理模块应该可以正常使用。

如果仍然报错，请检查：
1. 数据库连接配置是否正确
2. 数据库用户是否有创建表的权限
3. 数据库名称是否正确（应该是 `zcgl`）
