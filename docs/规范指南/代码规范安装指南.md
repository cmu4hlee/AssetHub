# 代码规范安装指南

## 快速开始

### 1. 安装依赖

**前端:**
```bash
cd frontend
npm install
```

**后端:**
```bash
cd backend
npm install
```

这将自动安装 ESLint 和 Prettier 相关依赖。

### 2. 验证安装

**前端:**
```bash
npm run lint          # 应该能看到 ESLint 检查结果
npm run format:check  # 检查代码格式
```

**后端:**
```bash
npm run lint          # 应该能看到 ESLint 检查结果
npm run format:check  # 检查代码格式
```

### 3. 格式化现有代码（可选）

如果你想格式化所有现有代码：

**前端:**
```bash
npm run format
```

**后端:**
```bash
npm run format
```

⚠️ **注意**: 这会修改所有代码文件，建议先提交当前更改或创建新分支。

## VS Code 用户

### 安装扩展

1. 打开 VS Code
2. 安装以下扩展：
   - **ESLint** (dbaeumer.vscode-eslint)
   - **Prettier - Code formatter** (esbenp.prettier-vscode)

### 配置已自动完成

项目根目录已包含 `.vscode/settings.json`，会自动：
- 保存时自动格式化
- 保存时自动修复 ESLint 问题
- 使用 Prettier 作为默认格式化工具

### 验证配置

1. 打开任意 `.js` 或 `.jsx` 文件
2. 右键 → "格式化文档" (或 Shift+Alt+F)
3. 应该看到代码被格式化

## 使用建议

### 日常开发

1. **保存时自动格式化**: VS Code 会在保存时自动格式化代码
2. **提交前检查**: 运行 `npm run lint` 确保没有错误
3. **修复问题**: 运行 `npm run lint:fix` 自动修复可修复的问题

### 团队协作

1. **统一配置**: 所有团队成员使用相同的 ESLint 和 Prettier 配置
2. **提交前格式化**: 建议在提交前运行 `npm run format`
3. **CI/CD 检查**: 可以在 CI/CD 流程中添加格式检查

## 常见问题

### Q: 安装依赖时出错

**A:** 确保 Node.js 版本 >= 16，然后：
```bash
rm -rf node_modules package-lock.json
npm install
```

### Q: ESLint 报错但代码看起来没问题

**A:** 
1. 运行 `npm run lint:fix` 自动修复
2. 查看具体错误信息，可能需要调整代码
3. 某些规则可以配置为警告而非错误

### Q: Prettier 格式化后代码变化很大

**A:** 这是正常的，Prettier 会统一代码风格。建议：
- 配置 IDE 自动格式化，避免手动格式化
- 团队统一使用相同的配置

### Q: 某些文件不想被格式化

**A:** 在 `.prettierignore` 或 `.eslintignore` 中添加文件路径。

### Q: 如何修改代码规范规则？

**A:** 
- ESLint: 修改 `.eslintrc.js` (后端) 或 `eslint.config.js` (前端)
- Prettier: 修改 `.prettierrc`

修改后运行 `npm run lint:fix` 和 `npm run format` 更新现有代码。
