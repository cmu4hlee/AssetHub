# 数据库凭据安全配置指南

## 问题说明

项目中存在硬编码的数据库凭据，这是一个严重的安全风险。硬编码的密码可能被提交到版本控制系统，导致密码泄露。

## 解决方案

### 1. 使用环境变量

所有数据库配置现在都支持通过环境变量设置，优先使用环境变量，避免硬编码。

### 2. 配置步骤

#### 步骤 1：安装 dotenv（如果尚未安装）

```bash
cd Asset/backend
npm install dotenv
```

#### 步骤 2：创建 .env 文件

在 `Asset/backend/` 目录下创建 `.env` 文件：

```bash
cp .env.example .env
```

#### 步骤 3：编辑 .env 文件

编辑 `.env` 文件，填入你的数据库配置：

```env
# 数据库配置
DB_HOST=101.37.236.101
DB_PORT=3306
DB_USER=root
DB_PASSWORD=你的实际密码
DB_NAME=zcgl

# 其他配置...
```

#### 步骤 4：确保 .env 文件不被提交

检查 `Asset/backend/.gitignore` 文件，确保包含：

```
.env
.env.local
.env.*.local
```

### 4. 使用方法

#### 运行脚本

所有脚本现在会自动加载 `.env` 文件：

```bash
# 初始化数据库
node scripts/init-db.js

# 检查用户
node check-users.js
```

#### 运行服务器

服务器启动时会自动加载 `.env` 文件（如果 `server.js` 中已配置 `dotenv`）。

### 5. 环境变量列表

| 变量名 | 说明 | 默认值 | 必需 |
|--------|------|--------|------|
| `DB_HOST` | 数据库主机地址 | `localhost` | 是 |
| `DB_PORT` | 数据库端口 | `3306` | 否 |
| `DB_USER` | 数据库用户名 | `root` | 是 |
| `DB_PASSWORD` | 数据库密码 | - | **是** |
| `DB_NAME` | 数据库名称 | `zcgl` | 是 |
| `DB_CONNECTION_LIMIT` | 连接池大小 | `10` | 否 |
| `DB_QUEUE_LIMIT` | 队列限制 | `0` | 否 |

### 6. 安全建议

1. **永远不要提交 .env 文件**
   - 确保 `.gitignore` 包含 `.env`
   - 在提交前检查是否有敏感信息

2. **使用不同的密码**
   - 开发环境、测试环境、生产环境使用不同的密码
   - 不要使用默认密码

3. **定期更换密码**
   - 定期更换数据库密码
   - 更新 `.env` 文件中的密码

4. **限制访问权限**
   - 数据库用户只授予必要的权限
   - 使用最小权限原则

5. **使用密钥管理服务（生产环境）**
   - 对于生产环境，考虑使用密钥管理服务（如 AWS Secrets Manager、Azure Key Vault）
   - 不要将生产环境密码存储在代码或配置文件中

### 7. 验证配置

运行以下命令验证配置是否正确：

```bash
# 测试数据库连接
node -e "require('dotenv').config(); const db = require('./config/database'); db.getConnection().then(conn => { conn.ping().then(() => { console.log('✅ 数据库连接成功'); conn.release(); process.exit(0); }); }).catch(err => { console.error('❌ 数据库连接失败:', err.message); process.exit(1); });"
```

### 8. 迁移现有配置

如果你之前使用硬编码配置，需要：

1. 创建 `.env` 文件
2. 将硬编码的值复制到 `.env` 文件
3. 确保 `.env` 在 `.gitignore` 中
4. 提交代码（不包含 `.env`）

### 9. 团队协作

对于团队开发：

1. 提交 `.env.example` 文件（包含示例值，不包含真实密码）
2. 每个开发者创建自己的 `.env` 文件
3. 在文档中说明如何配置 `.env` 文件

### 10. CI/CD 配置

在 CI/CD 环境中：

1. 使用 CI/CD 平台的环境变量功能
2. 不要将密码硬编码在 CI/CD 配置文件中
3. 使用密钥管理服务存储敏感信息
