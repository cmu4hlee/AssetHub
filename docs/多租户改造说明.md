# 多租户系统改造说明

## 概述

系统已改造为支持多租户（Multi-tenancy）架构，允许多个企业独立使用系统，数据完全隔离。

## 已完成的工作

### 1. 数据库层面

- ✅ 创建了 `tenants` 表（租户/企业表）
- ✅ 为所有业务表添加了 `tenant_id` 字段
- ✅ 为现有数据分配了默认租户
- ✅ 用户表已关联租户

### 2. 后端层面

- ✅ 创建了租户管理API (`/api/tenants`)
- ✅ 修改了认证中间件，添加租户验证
- ✅ 修改了登录接口，支持租户选择
- ✅ 创建了租户过滤辅助函数 (`middleware/tenant-filter.js`)
- ✅ 修改了部分接口示例（assets接口）

### 3. 功能特点

- **数据隔离**：每个租户的数据完全隔离
- **权限控制**：系统管理员可以管理所有租户，普通用户只能访问自己租户的数据
- **租户管理**：支持创建、更新、停用租户
- **订阅管理**：支持不同订阅类型和限制

## 如何修改其他接口

### 步骤1：导入租户过滤函数

在路由文件顶部添加：

```javascript
const { addTenantFilter, getTenantId } = require('../middleware/tenant-filter');
```

### 步骤2：在查询中添加租户过滤

对于 SELECT 查询，在 WHERE 子句构建后添加：

```javascript
let whereClause = 'WHERE 1=1';
const params = [];

// ... 其他过滤条件 ...

// 添加租户过滤
const tenantFilter = addTenantFilter(req, '表别名');
whereClause += tenantFilter.whereClause;
params.push(...tenantFilter.params);
```

### 步骤3：在插入时添加租户ID

对于 INSERT 语句，添加 `tenant_id` 字段：

```javascript
const tenantId = getTenantId(req);

await db.execute(
  'INSERT INTO table_name (tenant_id, field1, field2, ...) VALUES (?, ?, ?, ...)',
  [tenantId, value1, value2, ...]
);
```

### 步骤4：在更新时验证租户

对于 UPDATE 和 DELETE 操作，确保只能操作自己租户的数据：

```javascript
// 在更新前检查租户
const [existing] = await db.execute(
  'SELECT tenant_id FROM table_name WHERE id = ?',
  [id]
);

if (existing.length === 0) {
  return res.status(404).json({ success: false, message: '记录不存在' });
}

// 系统管理员可以操作所有租户的数据，普通用户只能操作自己租户的数据
if (req.user.role !== 'system_admin' && existing[0].tenant_id !== req.user.tenant_id) {
  return res.status(403).json({ success: false, message: '无权访问此数据' });
}
```

## 前端改造

### 1. 登录页面

需要在登录表单中添加租户选择：

```jsx
<Form.Item name="tenant_code" label="企业编码">
  <Input placeholder="请输入企业编码（可选）" />
</Form.Item>
```

### 2. 用户信息显示

在用户信息中显示当前租户：

```jsx
{user.tenant_name && (
  <div>当前企业：{user.tenant_name}</div>
)}
```

### 3. 租户管理页面

系统管理员可以访问租户管理页面，进行租户的创建、编辑、停用等操作。

## 注意事项

1. **数据迁移**：现有数据已自动分配到默认租户（tenant_code: 'default'）
2. **系统管理员**：系统管理员默认可以访问所有租户的数据，如需限制可以修改 `addTenantFilter` 函数
3. **外键约束**：所有业务表的 `tenant_id` 都设置了外键约束，删除租户时会级联删除相关数据
4. **性能考虑**：所有查询都添加了 `tenant_id` 索引，确保查询性能
