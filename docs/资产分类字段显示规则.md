# 资产分类字段显示规则详细说明

## 📋 概述

资产表单页面根据选择的**一级分类**和**二级分类**动态显示相应的字段。字段显示规则采用多级匹配策略，确保准确性和灵活性。

## 🎯 匹配优先级

字段显示规则按以下优先级匹配：

1. **精确编码匹配**（最高优先级）
2. **编码前缀匹配**
3. **名称关键词匹配**
4. **描述关键词匹配**
5. **一级分类默认配置**（最低优先级）

## 🔧 字段映射规则

### 1. 医疗设备类（YL开头）

**适用分类：**
- 医疗影像设备 (YL-01)
- 生命支持设备 (YL-02)
- 检验设备 (YL-03)
- 其他YL开头的二级分类

**显示字段：**
- ✅ 品牌 (brand)
- ✅ 型号 (model)
- ✅ 出厂编号 (serial_number)
- ✅ 规格参数 (specification)
- ✅ 存放位置 (location)
- ✅ 存放地点 (storage_location)
- ✅ 保修期（月）(warranty_period)
- ✅ 保修到期日 (warranty_end_date)

**匹配规则：**
- 编码精确匹配：`YL-01`, `YL-02`, `YL-03`
- 编码前缀匹配：`YL-*`（排除`YL`本身）
- 名称关键词：`影像`, `ct`, `mri`, `x光`, `生命支持`, `呼吸机`, `监护仪`, `检验`, `生化`, `血球`
- 描述关键词：`ct`, `mri`, `x光`

### 2. 普通设备类（PT开头，排除车辆）

**适用分类：**
- 办公设备 (PT-01)
- 其他PT开头的二级分类（排除PT-02车辆）

**显示字段：**
- ✅ 品牌 (brand)
- ✅ 型号 (model)
- ✅ 出厂编号 (serial_number)
- ✅ 规格参数 (specification)
- ✅ 存放位置 (location)
- ✅ 存放地点 (storage_location)

- ✅ 保修到期日 (warranty_end_date)

**匹配规则：**
- 编码精确匹配：`PT-01`

- 名称关键词：`办公设备`, `电脑`, `计算机`, `打印机`, `复印机`
- 描述关键词：`电脑`, `打印机`

**匹配规则：**
- 编码精确匹配：`PT-02`
- 名称关键词：`车辆`, `汽车`, `车`
- 描述关键词：`车辆`, `汽车`

### 4. 建筑物类（FC-01）

- 名称包含"建筑物"、"建筑"的分类（排除土地）

- 名称关键词：`建筑物`, `建筑`, `房产`（排除"土地"）
- 描述关键词：`建筑`, `房产`

## 🔄 字段显示逻辑流程

```
选择一级分类
    ↓
加载二级分类列表
    ↓
选择二级分类？
    ├─ 是 → 根据二级分类匹配字段规则
    │        ├─ 精确编码匹配
    │        ├─ 编码前缀匹配
    │        ├─ 名称关键词匹配
    │        └─ 描述关键词匹配
    │
    └─ 否 → 根据一级分类显示默认字段
             └─ 一级分类字段映射表
```

## 📝 通用字段（所有分类都显示）

无论选择什么分类，以下字段都会显示：

- 资产编号 (asset_code) **必填**
- 原始编码 (code, code2, code3)
- 资产名称 (asset_name) **必填**
- 一级分类 (category_id) **必填**
- 二级分类 (category_secondary_id) **必填**
- 购置日期 (purchase_date)
- 购置价格 (purchase_price)
- 当前价值 (current_value)
- 折旧方式 (depreciation_method)
- 折旧年限 (depreciation_years)
- 使用部门 (department)
- 单位 (unit)
- 责任人 (responsible_person)
- 资产状态 (status)
- 供应商 (supplier)
- 备注 (remark)
- 数据ID (data_id)
- 原始创建时间 (original_created_at)

## 🎨 自定义分类支持

系统支持用户自定义分类，字段显示规则会智能推断：

1. **编码前缀匹配**：如果自定义分类使用标准编码前缀（YL、PT、FC、BG），会自动匹配相应字段
2. **名称关键词匹配**：根据分类名称中的关键词自动匹配字段
3. **描述关键词匹配**：根据分类描述中的关键词补充匹配
4. **一级分类回退**：如果无法匹配，使用一级分类的默认字段配置

## 🔍 调试信息

在开发环境下，字段显示时会输出调试日志：
- 显示字段名称
- 匹配的分类信息（名称、编码）
- 匹配规则类型

## 📌 注意事项

1. **字段必填规则**：某些字段在特定分类下为必填（如车辆的车牌号码、土地建筑的产权证号）
2. **字段清理**：切换分类时，不相关的字段值会自动清空
3. **租户隔离**：分类数据按租户隔离，每个租户只能看到和管理自己的分类
4. **分类层级**：一级分类的`parent_id`为0，二级分类的`parent_id`指向一级分类的ID

## 🚀 扩展说明

如需添加新的分类字段规则，请在`getCategoryFields`函数中添加相应的匹配逻辑，遵循优先级顺序。
