## 🔧 修复方式

### 1. 使用统一的配置助手

所有脚本现在都使用 `backend/scripts/db-config-helper.js` 模块来获取数据库配置：

```javascript
// 修复前
const config = {
  host: '101.37.236.101',
  port: 3306,
  user: 'root',
  password: 'Cmu19801008',  // ❌ 硬编码
  database: 'zcgl',
};

// 修复后
const { getDatabaseConfig } = require('./db-config-helper');
const config = getDatabaseConfig();  // ✅ 从环境变量读取
```

### 2. 强制要求环境变量

`db-config-helper.js` 会检查 `DB_PASSWORD` 环境变量：

- ✅ 如果设置了环境变量，使用环境变量的值
- ❌ 如果未设置，脚本会报错并退出，提示用户设置环境变量

### 3. ServiceManager 修复

`ServiceManager/main.js` 中的硬编码凭据已移除：

```javascript
// 修复前
const dbConfig = {
  host: '101.37.236.101',
  password: 'Cmu19801008',  // ❌ 硬编码
  // ...
};

// 修复后
const dbConfig = {
  host: 'localhost',  // 默认值
  password: '',  // 不允许硬编码，必须从 .env 读取
  // ...
};
// 优先从 .env 文件读取配置
```

## 📝 使用方法

### 运行脚本前

1. **确保已创建 `.env` 文件**

在 `backend/` 目录下创建 `.env` 文件：

```env
DB_HOST=your_database_host
DB_PORT=3306
DB_USER=root
DB_PASSWORD=your_secure_password
DB_NAME=zcgl
```

2. **运行脚本**

```bash
cd backend
node scripts/your-script.js
```

如果未设置 `DB_PASSWORD`，脚本会显示错误信息并退出：

```
❌ 错误：数据库密码未设置！
请设置环境变量 DB_PASSWORD 或创建 .env 文件
示例：DB_PASSWORD=your_password

提示：可以复制 .env.example 为 .env 并填入实际配置
```

## 🔒 安全改进

### 修复前的问题

- ❌ 硬编码的数据库密码可能被提交到版本控制系统
- ❌ 密码泄露风险
- ❌ 不同环境使用相同密码

### 修复后的优势

- ✅ 所有凭据通过环境变量管理
- ✅ 密码不会出现在代码中
- ✅ 不同环境可以使用不同的配置
- ✅ 符合安全最佳实践
