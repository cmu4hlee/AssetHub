# .env 文件配置说明

## 📝 如何创建 .env 文件

### Windows 系统

1. 在 `backend` 目录下，右键 → 新建 → 文本文档
2. 将文件重命名为 `.env`（注意：文件名前面有点号）
3. 如果系统提示"如果改变文件扩展名，文件可能不可用"，点击"是"
4. 用记事本或代码编辑器打开 `.env` 文件
5. 复制下面的配置模板，根据实际情况修改

### 或者使用命令行

```cmd
cd C:\AssetManagement\backend
copy con .env
# 然后粘贴配置内容，按 Ctrl+Z 然后回车保存
```

## 🔧 完整配置模板

将以下内容复制到 `.env` 文件中，并根据实际情况修改：

```env
# ============================================
# 环境配置
# ============================================
# 运行环境: development（开发）或 production（生产）
NODE_ENV=production

# ============================================
# 数据库配置（必填）
# ============================================
# 数据库主机地址
DB_HOST=localhost

# 数据库端口（MySQL 默认 3306）
DB_PORT=3306

# 数据库用户名
DB_USER=root

# 数据库密码（必填，请修改为实际密码）
DB_PASSWORD=你的数据库密码

# 数据库名称
DB_NAME=zcgl

# 数据库连接池配置（可选，使用默认值即可）
# DB_CONNECTION_LIMIT=10
# DB_QUEUE_LIMIT=0
# DB_CONNECT_TIMEOUT=10000
# DB_IDLE_TIMEOUT=300000
# DB_MAX_IDLE=3600000

# ============================================
# 服务器配置
# ============================================
# 后端服务端口（默认 4001）
PORT=4001

# 服务器主机（0.0.0.0 表示监听所有网络接口）
SERVER_HOST=0.0.0.0

# ============================================
# JWT 配置（必填，生产环境）
# ============================================
# JWT 密钥（必填！请修改为随机字符串，建议至少 32 位）
# 生成随机密钥的方法：
# - Windows PowerShell: [Convert]::ToBase64String((1..32 | ForEach-Object { Get-Random -Minimum 0 -Maximum 256 }))
# - 在线工具: https://randomkeygen.com/
JWT_SECRET=请修改为随机密钥字符串至少32位

# Token 过期时间（秒，默认 7 天）
# JWT_EXPIRES_IN=604800

# Token 刷新时间（秒，默认 30 天）
# JWT_REFRESH_EXPIRES_IN=2592000

# ============================================
# CORS 跨域配置
# ============================================
# 允许的源（多个用逗号分隔，* 表示允许所有）
# 生产环境建议指定具体域名，例如: http://yourdomain.com,https://yourdomain.com
CORS_ORIGIN=*

# 是否允许携带凭证
# CORS_CREDENTIALS=false

# 预检请求缓存时间（秒）
# CORS_MAX_AGE=86400

# ============================================
# 前端配置
# ============================================
# 前端访问地址（如果使用后端提供前端，可以忽略）
# FRONTEND_URL=http://localhost:4000
# FRONTEND_DOMAIN=localhost
# FRONTEND_PORT=4000

# ============================================
# 网络访问控制（可选）
# ============================================
# 是否启用 IP 白名单（true/false）
# ENABLE_IP_WHITELIST=false

# 允许访问的 IP 列表（多个用逗号分隔）
# 例如: ALLOWED_IPS=192.168.1.100,10.0.0.0/8

# 是否启用域名白名单（true/false）
# ENABLE_DOMAIN_WHITELIST=false

# 允许访问的域名列表（多个用逗号分隔）
# 例如: ALLOWED_DOMAINS=example.com,*.example.com

# 是否信任代理（如果使用 Nginx 等反向代理，设置为 true）
# TRUST_PROXY=false

# 信任的代理 IP 列表
# TRUST_PROXY_IPS=127.0.0.1,::1

# ============================================
# 文件上传配置（可选）
# ============================================
# 上传文件存储目录（默认: backend/uploads）
# UPLOAD_DIR=./uploads

# 最大文件大小（字节，默认 10MB）
# MAX_FILE_SIZE=10485760

# 允许的文件类型（多个用逗号分隔）
# ALLOWED_MIME_TYPES=image/jpeg,image/png,image/gif,application/pdf

# 是否启用文件类型检查
# ENABLE_MIME_TYPE_CHECK=true

# ============================================
# 日志配置（可选）
# ============================================
# 日志级别: error, warn, info, debug
# LOG_LEVEL=warn

# 日志文件目录
# LOG_DIR=./logs

# 是否启用文件日志
# ENABLE_FILE_LOG=true

# 是否启用控制台日志
# ENABLE_CONSOLE_LOG=true

# 日志文件最大大小（字节，默认 10MB）
# MAX_LOG_FILE_SIZE=10485760

# 保留的日志文件数量
# MAX_LOG_FILES=10

# ============================================
# 备份配置（可选）
# ============================================
# 备份文件存储目录
# BACKUP_DIR=./backups

# 自动备份间隔（小时，0 表示禁用）
# AUTO_BACKUP_INTERVAL=0

# 保留的备份文件数量
# MAX_BACKUP_FILES=30

# 是否压缩备份文件
# COMPRESS_BACKUP=true

# ============================================
# HTTPS 配置（可选，如果使用 HTTPS）
# ============================================
# 是否启用 HTTPS
# HTTPS=false

# SSL 证书路径
# SSL_CERT=./certs/cert.pem
# SSL_KEY=./certs/key.pem
```

## 📋 配置示例

### 示例 1：本地开发环境

```env
NODE_ENV=development
DB_HOST=localhost
DB_PORT=3306
DB_USER=root
DB_PASSWORD=123456
DB_NAME=zcgl
PORT=4001
JWT_SECRET=development-secret-key-change-in-production
CORS_ORIGIN=*
```

### 示例 2：生产环境（单服务器）

```env
NODE_ENV=production
DB_HOST=localhost
DB_PORT=3306
DB_USER=root
DB_PASSWORD=你的强密码
DB_NAME=zcgl
PORT=4001
SERVER_HOST=0.0.0.0
JWT_SECRET=生成的32位以上随机密钥
CORS_ORIGIN=*
```

### 示例 3：生产环境（分离部署）

```env
NODE_ENV=production
DB_HOST=192.168.1.100
DB_PORT=3306
DB_USER=asset_user
DB_PASSWORD=强密码
DB_NAME=zcgl
PORT=4001
SERVER_HOST=0.0.0.0
JWT_SECRET=生成的32位以上随机密钥
CORS_ORIGIN=http://yourdomain.com,https://yourdomain.com
FRONTEND_URL=https://yourdomain.com
```

## 🔐 生成 JWT 密钥

### Windows PowerShell

```powershell
# 生成 32 字节随机密钥（Base64 编码）
[Convert]::ToBase64String((1..32 | ForEach-Object { Get-Random -Minimum 0 -Maximum 256 }))

# 或生成 64 字符随机字符串
-join ((65..90) + (97..122) + (48..57) | Get-Random -Count 64 | ForEach-Object {[char]$_})
```

### 在线工具

访问：https://randomkeygen.com/ 或 https://www.lastpass.com/features/password-generator

选择 "CodeIgniter Encryption Keys" 或生成至少 32 位的随机字符串。

## ✅ 配置检查清单

配置完成后，检查以下项：

- [ ] `DB_HOST` - 数据库主机地址正确
- [ ] `DB_PORT` - 数据库端口正确（MySQL 默认 3306）
- [ ] `DB_USER` - 数据库用户名正确
- [ ] `DB_PASSWORD` - 数据库密码正确（**必填**）
- [ ] `DB_NAME` - 数据库名称正确（通常是 `zcgl`）
- [ ] `JWT_SECRET` - JWT 密钥已修改（**生产环境必填**）
- [ ] `NODE_ENV` - 环境设置正确（开发用 `development`，生产用 `production`）
- [ ] `PORT` - 端口号未被占用

## 🧪 测试配置

配置完成后，启动服务测试：

```cmd
cd C:\AssetManagement\backend
node server.js
```

如果看到以下信息，说明配置正确：

```
✅ 数据库连接成功
✅ 服务器运行在 http://0.0.0.0:4001
```

如果出现错误，检查：

1. **数据库连接失败**
   - 检查 MySQL 服务是否运行
   - 检查数据库用户名和密码是否正确
   - 检查数据库是否存在

2. **端口被占用**
   ```cmd
   netstat -ano | findstr :4001
   taskkill /PID <进程ID> /F
   ```

3. **JWT 密钥未设置（生产环境）**
   - 确保 `JWT_SECRET` 已设置且不是默认值

## 📝 注意事项

1. **不要提交 .env 到 Git**
   - `.env` 文件包含敏感信息，不要提交到版本控制系统
   - 项目已配置 `.gitignore` 忽略 `.env` 文件

2. **生产环境安全**
   - 必须修改 `JWT_SECRET` 为随机字符串
   - 使用强密码作为数据库密码
   - 限制 `CORS_ORIGIN` 为具体域名，不要使用 `*`

3. **文件路径**
   - Windows 路径可以使用 `/` 或 `\`
   - 相对路径相对于 `backend` 目录

4. **注释**
   - `.env` 文件中 `#` 开头的行为注释
   - 注释不会影响配置

## 🔄 更新配置

```cmd
# 如果使用 PM2
pm2 restart asset-backend

# 如果使用 Windows 服务
net stop "AssetManagement Backend"
net start "AssetManagement Backend"

# 如果直接运行
# 按 Ctrl+C 停止，然后重新运行 node server.js
```
