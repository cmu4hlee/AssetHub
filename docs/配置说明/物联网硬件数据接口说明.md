# 物联网硬件数据接口说明

> **创建时间**: 2024-01-13  
> **用途**: 为外部物联网硬件设备提供数据上报接口

---

## 📡 API 接口

### 1. 硬件数据上报接口

**接口地址**: `POST /api/iot-devices/:deviceId/data`

**说明**: 外部物联网设备使用此接口上报数据，无需认证（通过设备ID验证）

**请求参数**:
- `deviceId` (路径参数): 设备唯一标识

**请求体** (JSON):
```json
{
  "latitude": 39.9042,           // 纬度（可选）
  "longitude": 116.4074,         // 经度（可选）
  "altitude": 50.5,              // 海拔高度，单位：米（可选）
  "signal_strength": 85,          // 信号强度，0-100（可选）
  "battery_level": 80,            // 电池电量，0-100（可选）
  "temperature": 25.5,            // 温度，单位：摄氏度（可选）
  "humidity": 60.5,               // 湿度，单位：%（可选）
  "other_data": {}                // 其他自定义数据（可选）
}
```

**响应示例**:
```json
{
  "success": true,
  "message": "数据上报成功",
  "data": {
    "device_id": "DEVICE001",
    "received_at": "2024-01-13T12:00:00.000Z"
  }
}
```

**功能说明**:
1. 验证设备是否存在
2. 更新设备状态为"在线"
3. 如果设备已关联资产，自动更新资产位置信息
4. 记录位置历史（如果提供了位置信息）

---

### 2. 获取设备数据接口

**接口地址**: `GET /api/iot-devices/:deviceId/data`

**说明**: 获取设备的历史数据（需要认证）

**请求参数**:
- `deviceId` (路径参数): 设备唯一标识
- `page` (查询参数): 页码，默认 1
- `pageSize` (查询参数): 每页数量，默认 50
- `start_date` (查询参数): 开始日期，格式：YYYY-MM-DD
- `end_date` (查询参数): 结束日期，格式：YYYY-MM-DD

**响应示例**:
```json
{
  "success": true,
  "data": [
    {
      "id": 1,
      "asset_id": 123,
      "device_id": "DEVICE001",
      "latitude": 39.9042,
      "longitude": 116.4074,
      "record_time": "2024-01-13T12:00:00.000Z",
      "update_source": "设备自动上报"
    }
  ],
  "pagination": {
    "page": 1,
    "pageSize": 50,
    "total": 100,
    "totalPages": 2
  }
}
```

## 🔧 使用示例

### Python 示例

```python
import requests
import json
import time

# 设备配置
DEVICE_ID = "DEVICE001"
API_BASE_URL = "http://your-server:4001"

def report_data(latitude, longitude, battery_level=100):
    """上报设备数据"""
    url = f"{API_BASE_URL}/api/iot-devices/{DEVICE_ID}/data"
    
    data = {
        "latitude": latitude,
        "longitude": longitude,
        "battery_level": battery_level,
        "signal_strength": 85
    }
    
    try:
        response = requests.post(url, json=data)
        result = response.json()
        if result.get("success"):
            print(f"数据上报成功: {result.get('message')}")
            return True
        else:
            print(f"数据上报失败: {result.get('message')}")
            return False
    except Exception as e:
        print(f"请求失败: {e}")
        return False

# 定时上报
while True:
    # 获取GPS位置（示例）
    lat = 39.9042
    lng = 116.4074
    
    report_data(lat, lng)
    time.sleep(60)  # 每60秒上报一次
```

### Node.js 示例

```javascript
const axios = require('axios');

const DEVICE_ID = 'DEVICE001';
const API_BASE_URL = 'http://your-server:4001';

async function reportData(latitude, longitude, batteryLevel = 100) {
  try {
    const response = await axios.post(
      `${API_BASE_URL}/api/iot-devices/${DEVICE_ID}/data`,
      {
        latitude,
        longitude,
        battery_level: batteryLevel,
        signal_strength: 85,
      }
    );

    if (response.data.success) {
      console.log('数据上报成功:', response.data.message);
      return true;
    } else {
      console.log('数据上报失败:', response.data.message);
      return false;
    }
  } catch (error) {
    console.error('请求失败:', error.message);
    return false;
  }
}

// 定时上报
setInterval(async () => {
  const lat = 39.9042;
  const lng = 116.4074;
  await reportData(lat, lng);
}, 60000); // 每60秒上报一次
```

**接口**: `POST /api/iot-devices` (需要认证)

```json
{
  "device_id": "DEVICE001",
```

## 🔒 安全说明

### 当前实现
- 数据上报接口通过设备ID验证设备存在性
- 管理接口需要用户认证

### 建议增强（可选）
1. **设备密钥验证**: 为每个设备分配密钥，上报时验证
2. **IP白名单**: 限制允许上报的IP地址
3. **频率限制**: 限制每个设备的上报频率
4. **数据加密**: 对敏感数据进行加密传输

## 📊 数据流程

```
外部设备
  ↓
POST /api/iot-devices/:deviceId/data
  ↓
验证设备存在
  ↓
更新设备状态为"在线"
  ↓
查找关联的资产
  ↓
更新资产位置信息
  ↓
记录位置历史
  ↓
返回成功响应
```

## 🐛 错误处理

### 设备不存在
```json
{
  "success": false,
  "message": "设备不存在"
}
```
**状态码**: 404

### 数据格式错误
```json
{
  "success": false,
  "message": "数据上报失败",
  "error": "错误详情"
}
```
**状态码**: 500

---

## 📝 注意事项

1. **设备ID必须唯一**: 在创建设备时确保设备ID不重复
2. **位置信息**: 如果提供位置信息，会自动更新关联资产的位置
3. **设备状态**: 每次上报数据会自动更新设备状态为"在线"
4. **数据存储**: 位置数据存储在 `asset_location_history` 表中
5. **关联资产**: 只有关联了资产的设备，位置数据才会更新到资产
