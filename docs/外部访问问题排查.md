# 外部访问前端问题排查指南

## 🔍 当前状态

**服务状态**：✅ 运行中（端口6000）  
**网络监听**：✅ 已监听所有接口（0.0.0.0:6000）  
**curl测试**：✅ 可以访问（返回200 OK）

## ⚠️ 如果浏览器无法访问

### 1. 清除浏览器缓存
- 按 `Cmd+Shift+R` (Mac) 或 `Ctrl+Shift+R` (Windows) 强制刷新
- 或使用无痕模式访问

### 2. 检查浏览器控制台
按 `F12` 打开开发者工具，查看 Console 和 Network 标签页的错误信息

### 3. 检查防火墙
**macOS**：
```bash
# 检查防火墙状态
sudo /usr/libexec/ApplicationFirewall/socketfilterfw --getglobalstate

# 如果防火墙开启，需要允许Node.js接收传入连接
# 系统设置 > 安全性与隐私 > 防火墙 > 选项
```

### 4. 尝试不同的访问方式

**方式1：使用localhost**
```
http://localhost:6000
```

**方式2：使用127.0.0.1**
```
http://127.0.0.1:6000
```

**方式3：使用IP地址**
```
http://192.168.1.80:6000
```

### 5. 检查服务日志
查看前端服务启动窗口，确认：
- 服务是否显示 "Network: http://192.168.1.80:6000/"
- 是否有错误信息

### 6. 重启服务
如果修改了配置，需要重启：
```bash
# 停止服务（Ctrl+C）
# 重新启动
cd Asset/frontend
npm run dev
```

## 🔧 快速测试命令

```bash
# 测试本地访问
curl http://localhost:6000

# 测试IP访问
curl http://192.168.1.80:6000

# 检查端口监听
lsof -i:6000
```

## 💡 建议

如果curl可以访问但浏览器不行，通常是：
1. 浏览器缓存问题 - 清除缓存或使用无痕模式
2. 防火墙阻止浏览器 - 检查防火墙设置
3. 浏览器扩展干扰 - 禁用扩展测试
