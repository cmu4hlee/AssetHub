# 响应式布局问题修复指南

**创建日期**: 2026-01-14  
**问题**: 浏览器尺寸缩小时，部分页面内容被隐藏

---

## 🔍 问题模式

### 常见问题模式

1. **布局断点缺失**
   - 使用 `<Col xs={24} lg={10}>` 时，中等尺寸（768px-991px）没有指定
   - 导致内容在中等尺寸时可能被隐藏

2. **高度设置问题**
   - Card 使用 `height: isMobile ? 'auto' : '100%'`
   - 在中等尺寸时，`isMobile` 为 false，但应该使用 `auto`

3. **溢出隐藏**
   - `overflow: 'hidden'` 导致内容被裁剪
   - 应该使用 `overflow: 'visible'`

---

## ✅ 修复方案

### 1. 添加平板尺寸检测

```javascript
// 在所有有布局问题的页面添加
const [isMobile, setIsMobile] = useState(window.innerWidth < 768);
const [isTablet, setIsTablet] = useState(window.innerWidth >= 768 && window.innerWidth < 992);

// 在 resize 处理中更新
useEffect(() => {
  const handleResize = () => {
    const width = window.innerWidth;
    setIsMobile(width < 768);
    setIsTablet(width >= 768 && width < 992);
    // 其他处理...
  };
  window.addEventListener('resize', handleResize);
  return () => window.removeEventListener('resize', handleResize);
}, []);
```

### 2. 修复 Card 高度设置

```javascript
// 修复前
style={{
  height: isMobile ? 'auto' : '100%',
}}

// 修复后
style={{
  height: isMobile || isTablet ? 'auto' : '100%',
  minHeight: isMobile || isTablet ? '400px' : 'auto',
  display: 'block',
}}
```

### 3. 修复 Card body 样式

```javascript
// 修复前
styles={{
  body: { padding: isMobile ? '12px' : '16px' },
}}

// 修复后
styles={{
  body: { 
    padding: isMobile ? '12px' : '16px',
    overflow: 'visible',
    minHeight: isMobile || isTablet ? '300px' : 'auto',
  },
}}
```

### 4. 修复 Descriptions 溢出

```javascript
// 修复前
<Descriptions
  column={isMobile ? 1 : 2}
  style={{
    overflow: 'hidden',
  }}
>

// 修复后
<Descriptions
  column={isMobile || isTablet ? 1 : 2}
  style={{
    overflow: 'visible',
    display: 'block',
    width: '100%',
  }}
>
```

### 5. 修复布局断点

```javascript
// 修复前
<Col xs={24} lg={10}>
<Col xs={24} lg={14}>

// 修复后（添加 md 断点）
<Col xs={24} md={24} lg={10}>
<Col xs={24} md={24} lg={14}>

// 或者使用更精确的断点
<Col xs={24} sm={24} md={24} lg={10}>
<Col xs={24} sm={24} md={24} lg={14}>
```

5. **测试验证**：在不同屏幕尺寸下测试

## 🎯 响应式断点参考

- **xs** (<576px): 超小屏幕（手机竖屏）
- **sm** (≥576px): 小屏幕（手机横屏）
- **md** (≥768px): 中等屏幕（平板）
- **lg** (≥992px): 大屏幕（桌面）
- **xl** (≥1200px): 超大屏幕
- **xxl** (≥1600px): 超超大屏幕

**关键断点**：
- 768px: 移动端/桌面端分界
- 992px: 平板/桌面端分界

---

## ⚠️ 注意事项

1. **地图容器**：地图容器的 `overflow: 'hidden'` 应该保留，这是正常的
2. **表格容器**：表格的 `overflow: 'hidden'` 可以保留，用于滚动
3. **内容容器**：内容区域的 `overflow` 应该使用 `visible`
