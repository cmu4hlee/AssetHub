## ✅ 建议的修复方案

### 方案1：使用环境变量（推荐）

#### 前端 vite.config.js
```javascript
import { defineConfig, loadEnv } from 'vite'

export default defineConfig(({ mode }) => {
  const env = loadEnv(mode, process.cwd(), '')
  
  return {
    server: {
      port: parseInt(env.VITE_FRONTEND_PORT) || 4000,
      proxy: {
        '/api': {
          target: env.VITE_BACKEND_URL || 'http://localhost:4001',
        }
      }
    },
    preview: {
      port: parseInt(env.VITE_PREVIEW_PORT) || 80,
      proxy: {
        '/api': {
          target: env.VITE_BACKEND_URL || 'http://localhost:4001',
        }
      }
    }
  }
})
```

#### 前端代码中使用环境变量
```javascript
// 创建 utils/config.js
export const config = {
  backendUrl: import.meta.env.VITE_BACKEND_URL || 'http://localhost:4001',
  frontendUrl: import.meta.env.VITE_FRONTEND_URL || 'http://localhost:4000',
}
```

### 方案2：使用相对路径（推荐用于生产环境）

对于文件URL，使用相对路径而不是绝对路径：
```javascript
// 之前
const fileUrl = `http://localhost:4000${file.file_path}`;

// 之后
const fileUrl = file.file_path; // 或 file.file_path.startsWith('/') ? file.file_path : `/${file.file_path}`;
```
