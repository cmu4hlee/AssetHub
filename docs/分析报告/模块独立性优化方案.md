# AssetHub 模块独立性优化方案

> **创建日期**: 2026-05-07
> **优化范围**: 模块解耦、服务层优化、依赖管理

---

## 📋 优化目标

根据[模块独立性分析报告](./模块独立性分析报告.md)，本次优化旨在：

1. **降低模块间耦合度** - 从紧耦合到松耦合
2. **统一依赖管理** - 使用依赖注入
3. **解耦模块通信** - 使用事件总线
4. **封装共享表访问** - 提供统一的数据访问接口

---

## 🎯 优化成果

### 1. 服务容器（依赖注入）

**文件位置**: `backend/core/ServiceContainer.js`

**功能**:
- 提供依赖注入容器
- 支持单例和工厂模式
- 服务解析和注册

**使用示例**:

```javascript
const { register, resolve } = require('../core/ServiceContainer');

// 注册服务（单例）
register('asset.service', new AssetService());

// 注册工厂（每次创建新实例）
registerFactory('user.service', () => new UserService());

// 解析服务
const assetService = resolve('asset.service');
```

**优势**:
- ✅ 降低模块间直接依赖
- ✅ 支持服务生命周期管理
- ✅ 便于单元测试（Mock）

---

### 2. 事件总线

**文件位置**: `backend/core/EventBus.js`

**功能**:
- 模块间松耦合通信
- 支持同步/异步事件
- 一次性事件订阅

**系统事件定义**:

```javascript
const { SYSTEM_EVENTS } = require('../core/EventBus');

// 资产事件
SYSTEM_EVENTS.ASSET_CREATED        // 资产创建
SYSTEM_EVENTS.ASSET_UPDATED        // 资产更新
SYSTEM_EVENTS.ASSET_STATUS_CHANGED // 资产状态变更
SYSTEM_EVENTS.ASSET_TRANSFERRED   // 资产调配

// 维修事件
SYSTEM_EVENTS.MAINTENANCE_CREATED   // 维修创建
SYSTEM_EVENTS.MAINTENANCE_COMPLETED // 维修完成

// 质量事件
SYSTEM_EVENTS.QUALITY_CONTROL_CREATED   // 质量控制创建
SYSTEM_EVENTS.QUALITY_CONTROL_COMPLETED  // 质量控制完成
SYSTEM_EVENTS.METROLOGY_EXPIRING         // 计量即将到期

// IoT事件
SYSTEM_EVENTS.IOT_DEVICE_ONLINE    // 设备上线
SYSTEM_EVENTS.IOT_DEVICE_OFFLINE   // 设备离线
SYSTEM_EVENTS.IOT_ALERT_TRIGGERED // 告警触发
```

**使用示例**:

```javascript
const { publish, subscribe } = require('../core/EventBus');

// 发布事件
await publish(SYSTEM_EVENTS.ASSET_STATUS_CHANGED, {
  assetCode: 'AST001',
  oldStatus: '维修中',
  newStatus: '在用',
  timestamp: new Date()
});

// 订阅事件
const unsubscribe = subscribe(SYSTEM_EVENTS.ASSET_STATUS_CHANGED, (event) => {
  console.log('Asset status changed:', event);
});

// 取消订阅
unsubscribe();
```

**优势**:
- ✅ 完全解耦模块间通信
- ✅ 支持异步处理
- ✅ 便于扩展和审计

### 3. 共享服务接口层

**文件位置**: `backend/services/shared/index.js`

**服务列表**:

| 服务 | 功能 | 适用范围 |
|------|------|---------|
| `AssetSharedService` | 资产共享服务 | 所有需要访问 assets 表的模块 |
| `UserSharedService` | 用户共享服务 | 所有需要访问 users 表的模块 |
| `DepartmentSharedService` | 部门共享服务 | 所有需要访问 departments 表的模块 |

**AssetSharedService API**:

```javascript
const { assetSharedService } = require('../services/shared');

// 根据资产编码获取资产
const asset = await assetSharedService.getAssetByCode('AST001', tenantId);

// 根据资产ID获取资产
const asset = await assetSharedService.getAssetById(123, tenantId);

// 获取资产状态
const status = await assetSharedService.getAssetStatus('AST001', tenantId);

// 更新资产状态
await assetSharedService.updateAssetStatus('AST001', '在用', tenantId, connection);

// 验证资产是否存在
const exists = await assetSharedService.assetExists('AST001', tenantId);

// 获取资产基本信息（轻量级）
const info = await assetSharedService.getAssetBasicInfo('AST001', tenantId);
```

**UserSharedService API**:

```javascript
const { userSharedService } = require('../services/shared');

// 根据用户ID获取用户
const user = await userSharedService.getUserById(1, tenantId);

// 根据用户名获取用户
const user = await userSharedService.getUserByUsername('admin', tenantId);

// 获取用户列表
const users = await userSharedService.getUsers(tenantId, {
  role: 'admin',
  department: 'IT',
  page: 1,
  pageSize: 50
});

// 验证用户是否存在
const exists = await userSharedService.userExists(1, tenantId);
```

**DepartmentSharedService API**:

```javascript
const { departmentSharedService } = require('../services/shared');

// 根据部门ID获取部门
const dept = await departmentSharedService.getDepartmentById(1, tenantId);

// 根据部门编码获取部门
const dept = await departmentSharedService.getDepartmentByCode('DEPT001', tenantId);

// 根据部门名称获取部门
const dept = await departmentSharedService.getDepartmentByName('技术部', tenantId);

// 获取部门列表
const depts = await departmentSharedService.getDepartments(tenantId, {
  parentId: 0,
  page: 1,
  pageSize: 50
});

// 获取部门树结构
const tree = await departmentSharedService.getDepartmentTree(tenantId);
```

**优势**:
- ✅ 封装共享表访问逻辑
- ✅ 统一数据访问接口
- ✅ 便于权限控制和审计
- ✅ 支持事务集成

### 4. 模块初始化管理器

**文件位置**: `backend/core/ModuleManager.js`

**功能**:
- 模块注册和初始化
- 服务自动注册
- 事件订阅管理
- 模块生命周期管理

**使用示例**:

```javascript
const { registerModule, initializeModules } = require('../core/ModuleManager');

// 注册模块
registerModule('maintenance-management', {
  name: '维护管理',
  services: [
    { name: 'maintenance', path: 'services/maintenance.service' },
    { name: 'requests', path: 'services/requests.service' }
  ],
  eventHandlers: [
    { event: 'asset:status-changed', callback: handleAssetStatusChange },
    { event: 'user:login', callback: handleUserLogin }
  ],
  onDestroy: async () => {
    console.log('Module destroyed');
  }
});

// 初始化所有模块
await initializeModules();
```

**优势**:
- ✅ 自动化模块初始化
- ✅ 统一服务注册
- ✅ 事件订阅管理
- ✅ 支持模块热卸载

## 🔄 重构指南

### 场景1: 跨模块服务调用

**问题代码（紧耦合）**:

```javascript
// maintenance/routes.js
const qualityCommonService = require('../../quality-common/services/quality-common.service');

router.post('/', async (req, res) => {
  // 直接依赖其他模块的服务
  await qualityCommonService.logQualityEvent(data);
  // ...
});
```

**重构后（松耦合）**:

```javascript
// maintenance/routes.js
const { resolve } = require('../../core/ServiceContainer');
const { publish, SYSTEM_EVENTS } = require('../../core/EventBus');

router.post('/', async (req, res) => {
  // 方式1: 使用服务容器
  const qualityCommonService = resolve('quality-common.service');
  await qualityCommonService.logQualityEvent(data);

  // 方式2: 使用事件总线（推荐）
  await publish(SYSTEM_EVENTS.QUALITY_CONTROL_CREATED, {
    type: 'maintenance',
    data
  });
  // ...
});
```

**优化前**:

## 📝 使用规范

### 1. 服务注册规范

```javascript
// ✅ 正确：在模块初始化时注册
register('module-name.service', new Service());

// ❌ 错误：在请求处理中注册
router.get('/', async (req, res) => {
  register('temp.service', new Service()); // 不要这样做
});
```

### 2. 事件命名规范

```javascript
// ✅ 正确：使用有意义的事件名
publish('asset:status-changed', data);
publish('maintenance:request-completed', data);

// ❌ 错误：使用模糊的事件名
publish('update', data);
publish('done', data);
```

### 3. 共享服务使用规范

```javascript
// ✅ 正确：使用共享服务
const asset = await assetSharedService.getAssetByCode(code, tenantId);

// ❌ 错误：直接查询共享表
const [rows] = await db.execute(
  'SELECT * FROM assets WHERE asset_code = ?',
  [code]
);
```

### 4. 事务处理规范

```javascript
// ✅ 正确：使用共享服务的事务支持
await assetSharedService.updateAssetStatus(code, status, tenantId, connection);

// ❌ 错误：混合使用事务和非事务查询
await connection.execute('UPDATE assets SET status = ?', [status]);
await db.execute('INSERT INTO logs VALUES (?)', [log]); // 没有事务！
```

## ⚠️ 注意事项

### 1. 事件处理的错误处理

```javascript
// ✅ 推荐：使用 try-catch 包装事件处理
subscribe(SYSTEM_EVENTS.ASSET_CREATED, async (event) => {
  try {
    await processAssetCreation(event);
  } catch (error) {
    logger.error('Failed to process asset creation:', error);
    // 不要在事件处理中抛出异常，可能导致事件总线崩溃
  }
});
```

### 2. 避免事件循环

```javascript
// ❌ 不要这样做：可能导致事件循环
subscribe(SYSTEM_EVENTS.ASSET_STATUS_CHANGED, async (event) => {
  // 改变状态后又发布相同事件
  await publish(SYSTEM_EVENTS.ASSET_STATUS_CHANGED, { /* ... */ });
});
```

### 3. 性能考虑

```javascript
// ✅ 推荐：使用一次性事件处理不需要长期监听的事件
subscribeOnce(SYSTEM_EVENTS.MODULE_ENABLED, (event) => {
  console.log('Module enabled:', event.moduleId);
});
```

## 🔧 故障排查

### 问题1: 服务未找到

```
Error: Service not found: xxx.service
```

**解决方案**:
1. 检查服务是否已注册
2. 检查服务名称是否正确
3. 检查注册时机（应在模块初始化时）

### 问题2: 事件未触发

**解决方案**:
1. 检查事件名称是否正确
2. 检查订阅时机（在发布前还是后）
3. 检查事件处理器是否抛出异常

### 问题3: 共享表访问失败

**解决方案**:
1. 检查 tenantId 是否正确传递
2. 检查数据是否存在
3. 检查权限是否足够
