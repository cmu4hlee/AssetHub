## 📊 权限系统总览

AssetHub 采用 **RBAC（基于角色的访问控制）** 模型，支持多租户、多角色、细粒度权限控制。

### 系统架构

```

│  │  生成/验证   │  │  独立表验证  │  │  动态租户ID  │           │
```

## 🔐 认证系统

### 1. JWT 认证

**文件**: [backend/middleware/auth.js](file:///Volumes/移动硬盘（500）/AssetHub/backend/middleware/auth.js)

**功能**:
- JWT Token 生成和验证
- Token 过期处理
- 多租户支持（通过 X-Tenant-ID 头）

**Token 内容**:
```javascript
{
  userId: user.id,
  username: user.username,
  role: user.role,           // 当前租户中的角色
  departmentCode: user.department_code,
}
```

**认证流程**:
1. 从 Authorization 头提取 Bearer Token
2. 验证 JWT 签名
3. 查询用户（超级用户表 / 普通用户表）
4. 处理租户切换（X-Tenant-ID）
5. 加载用户角色和权限
6. 检查模块访问权限

---

### 2. 用户类型

| 用户类型 | 表名 | 特点 | 权限 |
|---------|------|------|------|
| **超级管理员** | `super_users` | 全局管理，跨租户 | 所有权限 |
| **系统管理员** | `users` + `user_tenant_roles` | 租户级管理 | 租户内所有权限 |
| **普通用户** | `users` + `user_tenant_roles` | 业务操作 | 按角色分配权限 |

---

### 3. 租户切换机制

**支持动态租户切换**:
```javascript
// 通过 X-Tenant-ID 头切换租户
const dynamicTenantId = req.header('X-Tenant-ID');

// 验证用户是否有权访问该租户
const [tenantRoles] = await db.execute(
  `SELECT ur.tenant_id, ur.role
   FROM user_tenant_roles ur
   WHERE ur.user_id = ? AND ur.tenant_id = ? AND ur.status = ?`,
  [user.id, requestedTenantId, 'active']
);
```

## 📝 权限配置

### 1. 数据库表结构

#### 角色权限表 (role_permissions)

```sql
CREATE TABLE role_permissions (
  id INT PRIMARY KEY AUTO_INCREMENT,
  role VARCHAR(50) NOT NULL COMMENT '角色代码',
  permission VARCHAR(100) NOT NULL COMMENT '权限代码',
  tenant_id INT NULL COMMENT '租户ID（NULL表示全局）',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  UNIQUE KEY uk_role_permission_tenant (role, permission, tenant_id)
);
```

#### 用户租户角色表 (user_tenant_roles)

```sql
CREATE TABLE user_tenant_roles (
  id INT PRIMARY KEY AUTO_INCREMENT,
  user_id INT NOT NULL COMMENT '用户ID',
  tenant_id INT NOT NULL COMMENT '租户ID',
  role VARCHAR(50) NOT NULL COMMENT '角色代码',
  is_default TINYINT(1) DEFAULT 0 COMMENT '是否默认租户',
  status VARCHAR(20) DEFAULT 'active' COMMENT '状态',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### 用户管理科室表 (user_managed_departments)

```sql
CREATE TABLE user_managed_departments (
  id INT PRIMARY KEY AUTO_INCREMENT,
  user_id INT NOT NULL COMMENT '用户ID',
  tenant_id INT NOT NULL COMMENT '租户ID',
  department_code VARCHAR(50) NOT NULL COMMENT '科室编码',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### 租户模块配置表 (tenant_module_configs)

```sql
CREATE TABLE tenant_module_configs (
  id INT PRIMARY KEY AUTO_INCREMENT,
  tenant_id INT NOT NULL COMMENT '租户ID',
  module_id VARCHAR(50) NOT NULL COMMENT '模块ID',
  enabled TINYINT(1) DEFAULT 1 COMMENT '是否启用',
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

## 🎯 使用示例

### 后端使用

```javascript
const { authenticate, authorize, restrictToOwnDepartment } = require('../middleware/auth');

// 基础认证
router.get('/assets', authenticate, async (req, res) => {
  // 所有认证用户可访问
});

// 权限检查
router.post('/assets', authenticate, authorize('asset.add'), async (req, res) => {
  // 只有拥有 asset.add 权限的用户可访问
});

// 数据范围控制
router.put('/assets/:id', authenticate, authorize('asset.edit_own_department'), restrictToOwnDepartment, async (req, res) => {
  // 只能编辑本科室的资产
});

// 系统管理员权限
router.post('/backup', authenticate, requireSystemAdmin, async (req, res) => {
  // 只有系统管理员可访问
});
```

### 前端使用

```javascript
import { isSuperAdmin, isAdminRole } from '../utils/roleUtils';

// 角色检查
if (isSuperAdmin(user)) {
  // 超级管理员逻辑

if (isAdminRole(user.role)) {
  // 管理员逻辑
}

// 权限检查（通过后端API返回的权限列表）

  return userPermissions.includes(permission);
```

## 📞 总结

AssetHub 的权限系统具有以下特点：

1. **完善的RBAC模型** - 支持多角色、多租户、细粒度权限
2. **高效的缓存机制** - 5分钟TTL，LRU淘汰策略
3. **灵活的数据范围** - 全部/科室/个人三级数据隔离
4. **模块级访问控制** - 支持按模块启用/禁用
5. **完整的审计追踪** - 自动记录所有操作
6. **多重安全保护** - IP白名单、速率限制、高风险操作保护

建议后续优化：
- 将硬编码权限配置外部化
- 增加权限继承机制
- 完善前端权限管理
- 提供权限管理API
