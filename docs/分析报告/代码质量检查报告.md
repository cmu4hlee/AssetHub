## 🔴 高优先级问题

### 1. 前端未使用的导入和变量

**问题描述**：App.jsx 存在大量未使用的导入和变量声明

**影响范围**：
- `/frontend/src/App.jsx` - 约 60+ 未使用导入
- 所有 lazy 导入的组件都被赋值但未使用

**示例代码**：
```jsx
// App.jsx:1-8
import React, { Suspense, lazy } from 'react';  // React, Suspense 未使用
import { BrowserRouter as Router, Routes, Route, Navigate } from 'react-router-dom';  // 全部未使用
import ProtectedRoute from './components/ProtectedRoute';  // 未使用
import { DepartmentProvider } from './contexts/DepartmentContext';  // 未使用
```

**建议**：
1. 运行 `npm run lint:fix` 自动修复
2. 检查路由配置，确认是否需要这些组件

### 2. 代码重复 - catch 块过多

**问题描述**：项目中存在大量空的或简化的 catch 块

**统计数据**：
- 后端：492 个 catch 块（100 个文件）
- 前端：375 个 catch 块（100+ 个文件）

**问题示例**：
```javascript
// 缺少错误处理的 catch 块
try {
  await doSomething();
} catch (error) {
  // 空 catch 块，不处理错误
}

// 或者只记录日志
try {
  await doSomething();
} catch (error) {
  console.error(error);  // 缺少上下文信息
}
```

**建议**：
1. 统一错误处理模式
2. 添加错误上下文信息
3. 考虑使用统一错误处理中间件

### 3. console.log 过多

**问题描述**：项目中存在大量 console 语句

**统计数据**：
- 后端：1782 处 console 调用
- 前端：大量分布在前端各个文件

**建议**：
1. 使用统一的日志系统（如 Winston）
2. 在生产环境中禁用 console.log
3. 使用环境变量控制日志级别

## 🟡 中优先级问题

### 4. LabelPreview 组件使用 innerHTML

**文件位置**：`/frontend/src/components/LabelPreview.jsx`

**问题代码**：
```javascript
// 标签打印预览组件直接操作 DOM
const svgElement = document.getElementById(`label-preview-${id}`);
if (svgElement) {
  svgElement.innerHTML = svgContent;  // 存在 XSS 风险
}
```

**建议**：
1. 使用 DOMPurify 净化 HTML
2. 考虑使用 React 的 ref API 替代直接 DOM 操作

### 5. dangerouslySetInnerHTML 使用

**发现位置**：
- `/frontend/src/pages/Introduction.jsx:74` - CSS 动画样式
- `/frontend/src/pages/AssetAIAnalysis.jsx:38` - 注释说明已避免

**建议**：
1. 对于 CSS 动画，考虑使用 style 标签或 CSS-in-JS
2. Markdown 渲染已使用 react-markdown，应保持

### 6. TODO/FIXME 注释未处理

**统计数据**：40 处 TODO/FIXME/HACK 注释

**建议**：
1. 定期 review 并处理这些注释
2. 创建 issue 跟踪这些问题
