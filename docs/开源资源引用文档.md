## 定制化调整说明

### 1. 维护计划模板系统

**引用来源**: OpenMAINT、CMMS-OpenSource
**定制化调整**:
- 适配多租户架构，添加tenant_id字段
- 优化模板推荐算法，基于资产类型、品牌、型号进行智能匹配
- 增强JSON格式的维护项目和物料清单管理
- 集成到现有维护计划创建流程中

### 2. 工单管理系统

**引用来源**: CMMS-OpenSource、eam-open-source/eam
**定制化调整**:
- 适配多租户架构，确保数据隔离
- 优化工单状态流转，符合企业实际业务流程
- 集成库存管理，实现工单物料自动扣减
- 增强工单成本跟踪和分析功能

### 3. 库存管理模块

**引用来源**: OpenMAINT、eam-open-source/eam
**定制化调整**:
- 适配多租户架构，添加tenant_id字段
- 优化库存预警机制，基于最小库存和安全库存
- 增强库存变动记录和追溯功能
- 集成到维护物料需求管理中

### 4. 成本跟踪系统

**引用来源**: CMMS-OpenSource
**定制化调整**:
- 适配多租户架构，确保成本数据隔离
- 增强成本分类和分析功能
- 集成到维护日志和工单管理中
- 优化成本汇总和报表功能

## 版本跟踪机制

为确保及时获取安全更新与功能改进，建立以下版本跟踪机制：

1. **定期检查**:
   - 每季度检查一次引用开源项目的更新
   - 记录更新的版本号和主要变更

2. **变更评估**:
   - 评估更新对当前项目的影响
   - 制定更新计划和测试方案

3. **更新流程**:
   - 在测试环境验证更新
   - 确保更新不破坏现有功能
   - 记录更新过程和结果

4. **回滚机制**:
   - 建立更新失败的回滚方案
   - 确保系统稳定性

## 合规性声明

所有引用的开源资源均遵循各自的开源协议，本项目在使用过程中：

1. **遵守开源协议**:
   - 保留所有引用资源的版权声明
   - 遵循协议要求的 attribution 义务
   - 确保衍生作品的合规性

2. **责任限制**:
   - 本项目使用开源资源时，遵循"按原样"使用原则
   - 不承担因使用开源资源而产生的任何责任
   - 确保在文档中明确说明这一点

3. **安全性**:
   - 定期检查引用资源的安全漏洞
   - 及时应用安全补丁
   - 确保系统安全性
