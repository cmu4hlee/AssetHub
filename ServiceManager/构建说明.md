# 服务管理器 - 构建可执行程序说明

## 📦 macOS 应用构建

### 方法一：使用构建脚本（推荐）

1. **双击运行** `构建macOS应用.sh`
   - 首次运行会自动安装依赖
   - 构建完成后会在 `dist` 目录生成 `.dmg` 安装包

2. **安装应用**
   - 双击生成的 `.dmg` 文件
   - 将应用拖拽到"应用程序"文件夹
   - 在启动台中找到"资产管理服务管理器"并运行

### 方法二：命令行构建

```bash
cd ServiceManager
npm install
npm run build
```

### 快速启动（开发模式）

双击 `启动macOS应用.sh` 即可快速启动应用（开发模式，无需构建）

---

## 🪟 Windows 应用构建（Electron）

### 方法一：使用构建脚本（推荐）

1. **双击运行** `构建Windows应用.bat`
   - 首次运行会自动安装依赖
   - 构建完成后会在 `dist-new` 目录生成安装程序和便携版

2. **构建产物**
   - **安装程序**：`dist-new\资产管理服务管理器 Setup *.exe`（NSIS 安装程序）
     - 支持自定义安装目录
     - 自动创建桌面快捷方式
     - 自动创建开始菜单快捷方式
   - **便携版**：`dist-new\资产管理服务管理器 *.exe`（无需安装，直接运行）
   - **解压版**：`dist-new\win-unpacked\资产管理服务管理器.exe`（完整应用文件夹）

3. **使用应用**
   - **安装版**：双击安装程序，按照向导完成安装
   - **便携版**：直接双击 exe 文件运行
   - **解压版**：进入 `win-unpacked` 目录，双击 exe 运行

### 方法二：命令行构建

```cmd
cd ServiceManager
npm install
npm run build:win
```

### 快速启动（开发模式）

双击 `启动Windows应用.bat` 即可快速启动应用（开发模式，无需构建）
- 如果未安装依赖，会自动安装
- 使用 Electron 开发模式运行

---

## 🚀 使用已构建的应用

### macOS

1. 打开 `dist` 目录
2. 双击 `.dmg` 文件
3. 将应用拖到"应用程序"文件夹
4. 在启动台中找到并运行

### Windows

1. **安装版**：运行 `dist-new\资产管理服务管理器 Setup *.exe` 进行安装
2. **便携版**：直接运行 `dist-new\资产管理服务管理器 *.exe`
3. **解压版**：进入 `dist-new\win-unpacked` 目录，运行 `资产管理服务管理器.exe`
4. 可以将便携版或解压版复制到任何位置使用

## 📋 系统要求

### macOS
- macOS 10.13 或更高版本
- 已安装 Node.js（构建时需要）

### Windows
- Windows 7 或更高版本
- 已安装 Node.js（构建时需要）
- 已安装 npm（通常随 Node.js 一起安装）

---

## ⚠️ 注意事项

1. **首次运行**：应用会自动从项目文件中读取数据库配置
2. **配置文件**：应用会在运行目录创建 `config.json` 保存配置
3. **服务路径**：应用会自动查找项目根目录（包含 backend 和 frontend 文件夹的目录）
4. **端口占用**：确保配置的端口未被其他程序占用

---

## 🔧 故障排除

### macOS 构建失败
- 确保已安装 Node.js: `node --version`
- 确保已安装依赖: `npm install`
- 检查磁盘空间是否充足

### Windows 构建失败
- 确保已安装 Node.js: `node --version`
- 确保已安装依赖: `npm install`
- 检查磁盘空间是否充足
- 如果网络问题，脚本会自动使用国内镜像源
- 如果缺少图标文件，可以创建 `icon.ico` 文件（可选）

### 应用无法启动
- 检查是否有错误提示
- 查看控制台输出
- 确保项目目录结构正确

---

## 📞 技术支持

如有问题，请检查：
1. 项目目录结构是否正确
2. 依赖是否完整安装
3. 端口是否被占用
4. 配置文件是否正确
