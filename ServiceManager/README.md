# 资产管理系统服务管理器

一个基于 Electron 的 macOS 窗口应用，用于管理资产管理系统的前后端服务。

## 功能特性

- ✅ 启动/停止/重启前后端服务
- ✅ 实时服务状态监控
- ✅ 服务日志实时显示
- ✅ 配置管理（IP地址、端口号）
- ✅ 一键启动/停止所有服务
- ✅ 美观的现代化界面

## 安装

1. 进入 ServiceManager 目录：
```bash
cd ServiceManager
```

2. 安装依赖：
```bash
npm install
```

## 运行

### 开发模式
```bash
npm start
```

### 开发模式（带调试工具）
```bash
npm run dev
```

## 构建应用

构建 macOS 应用包：
```bash
npm run build
```

构建完成后，会在 `dist` 目录生成 `.dmg` 安装包。

## 使用说明

### 配置服务

1. 在"服务配置"区域设置：
   - 后端端口（生产模式默认：4001，开发模式建议：6001）
   - 后端主机（默认：0.0.0.0）
   - 前端端口（生产模式默认：4000，开发模式建议：6000）
   - 前端主机（默认：0.0.0.0）
   
   **端口说明**：
   - **开发模式**：前端6000，后端6001
   - **生产模式**：前端4000，后端4001

2. 点击"保存配置"按钮保存设置

### 控制服务

- **启动服务**：点击对应服务的"启动"按钮
- **停止服务**：点击对应服务的"停止"按钮
- **重启服务**：点击对应服务的"重启"按钮
- **快速操作**：
  - 启动所有服务：一键启动前后端
  - 停止所有服务：一键停止前后端
  - 重启所有服务：一键重启前后端

### 查看日志

每个服务下方都有日志输出区域，实时显示服务的运行日志。

### 服务状态

- 🟢 绿色圆点：服务运行中
- 🔴 红色圆点：服务已停止

## 配置文件

配置文件保存在 `ServiceManager/config.json`，格式如下：

```json
{
  "backend": {
    "port": 4001,
    "host": "0.0.0.0"
  },
  "frontend": {
    "port": 4000,
    "host": "0.0.0.0"
  }
}
```

**端口配置说明**：
- 默认配置为生产模式端口（4000/4001）
- 开发模式建议配置为：前端6000，后端6001
- 可在界面中直接修改端口配置

## 注意事项

1. 确保已安装 Node.js（推荐 v16 或更高版本）
2. 确保前后端目录存在且包含 `package.json`
3. 确保端口未被其他程序占用
4. 关闭应用时会自动停止所有服务

## 故障排除

### 服务启动失败
- 检查端口是否被占用
- 检查前后端目录路径是否正确
- 查看日志输出了解具体错误

### 无法保存配置
- 检查 ServiceManager 目录的写入权限
- 确保有足够的磁盘空间
