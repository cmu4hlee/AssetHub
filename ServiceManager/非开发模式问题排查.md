# 非开发模式功能问题排查

## 常见问题

### 1. 路径解析问题

**问题**: 打包后的应用无法找到 backend 和 frontend 目录

**原因**: 
- 打包后的应用路径与开发环境不同
- `__dirname` 在打包后指向 `app.asar` 内部
- 需要从应用安装位置向上查找项目根目录

**已修复**:
- ✅ 改进了 `getProjectRoot()` 函数
- ✅ 添加了多种路径查找策略
- ✅ 添加了详细的日志输出
- ✅ 支持从应用安装位置向上查找

### 2. 文件加载问题

**问题**: `index.html` 或 `renderer.js` 无法加载

**原因**:
- 打包后的文件路径可能不同
- 文件可能未正确打包到 `app.asar` 中

**已修复**:
- ✅ 改进了文件路径解析
- ✅ 添加了备用路径查找
- ✅ 添加了文件存在性检查

### 3. 资源文件问题

**问题**: CSS、图片等资源无法加载

**检查**:
- 确保 `package.json` 中的 `files` 配置包含所有需要的文件
- 检查 `asarUnpack` 配置是否正确

## 调试步骤

### 1. 查看控制台日志

打包后的应用会输出详细的路径查找日志：
- `检测到打包后的应用，查找项目根目录...`
- `app.getAppPath(): ...`
- `尝试查找项目根目录，可能的路径: ...`
- `✅ 找到项目根目录: ...`

### 2. 检查路径配置

在应用中点击"检查路径配置"按钮，查看：
- 项目根目录是否正确
- backend 和 frontend 目录是否存在
- server.js 和 package.json 是否存在

### 3. 验证文件位置

确保项目结构正确：
```
资产管理vite/
├── backend/          # 后端目录
│   └── server.js
├── frontend/         # 前端目录
│   └── package.json
└── ServiceManager/   # Electron 应用
    └── dist-new/     # 构建产物
```

## 解决方案

### 方案 1: 确保项目在正确位置

打包后的应用会尝试在以下位置查找项目：
1. 应用安装位置的父目录
2. `~/Desktop/Mac_Win_share/资产管理系统/资产管理vite`
3. `~/Desktop/资产管理系统/资产管理vite`
4. `~/Documents/Mac_Win_share/资产管理系统/资产管理vite`
5. `~/Documents/资产管理系统/资产管理vite`
6. 当前工作目录

**建议**: 将项目放在 `~/Desktop/Mac_Win_share/资产管理系统/资产管理vite`

### 方案 2: 使用环境变量

可以在启动应用前设置环境变量：
```bash
export PROJECT_ROOT=/path/to/项目根目录
```

### 方案 3: 在应用中配置路径

如果自动查找失败，可以在应用界面中手动配置路径（如果已实现此功能）。
