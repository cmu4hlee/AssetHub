# 资产管理系统服务管理器 - 使用说明

## 概述

这是一个 Windows Forms 应用程序，用于方便地管理资产管理系统的前端和后端服务。通过图形界面，您可以：

- 启动、停止、重启前端和后端服务
- 配置所有服务的 IP 地址和端口号
- 配置数据库连接信息

## 安装要求

1. **.NET 6.0 或更高版本**
   - 下载地址：https://dotnet.microsoft.com/download
   - 安装后可通过 `dotnet --version` 验证

2. **Node.js 和 npm**
   - 确保已安装 Node.js（建议 16.x 或更高版本）
   - 下载地址：https://nodejs.org/

3. **项目依赖**
   - 确保 `backend` 和 `frontend` 目录已安装依赖（运行过 `npm install`）

## 编译和运行

### 方法一：直接运行（开发模式）

1. 打开命令提示符或 PowerShell
2. 进入 `ServiceManager` 目录
3. 运行：
   ```bash
   dotnet run
   ```

或者双击 `启动服务管理器.bat`

### 方法二：编译后运行

1. 双击 `编译服务管理器.bat` 进行编译
2. 编译成功后，可执行文件位于：`bin\Release\net6.0-windows\ServiceManager.exe`
3. 双击 `ServiceManager.exe` 运行

## 使用步骤

### 1. 首次配置

1. 启动应用程序
2. 切换到"配置管理"标签页
3. 配置以下信息：

   **前端配置：**
   - 前端主机：通常为 `0.0.0.0`（允许所有 IP 访问）或 `localhost`（仅本地访问）
   - 前端端口：
     - 生产模式：默认 `4000`
     - 开发模式：建议 `6000`
   - API 地址：前端访问后端的 API 地址
     - 生产模式：`http://localhost:4001/api`
     - 开发模式：`http://localhost:6001/api`

   **后端配置：**
   - 后端主机：通常为 `0.0.0.0`（允许所有 IP 访问）或 `localhost`（仅本地访问）
   - 后端端口：
     - 生产模式：默认 `4001`
     - 开发模式：建议 `6001`

   **数据库配置：**
   - 数据库主机：数据库服务器的 IP 地址或域名
   - 数据库端口：MySQL 默认端口为 `3306`
   - 数据库名称：要连接的数据库名称
   - 用户名：数据库用户名
   - 密码：数据库密码

4. 点击"保存配置"按钮

### 2. 启动服务

1. 切换到"服务控制"标签页
2. 点击"后端服务"区域的"启动"按钮
3. 等待后端服务启动（状态显示为"运行中"）
4. 点击"前端服务"区域的"启动"按钮
5. 等待前端服务启动（状态显示为"运行中"）

### 3. 访问系统

**开发模式**：
- 前端地址：`http://localhost:6000`（或您配置的 IP 和端口）
- 后端 API：`http://localhost:6001/api`（或您配置的 IP 和端口）

**生产模式**：
- 前端地址：`http://localhost:4000`（或您配置的 IP 和端口）
- 后端 API：`http://localhost:4001/api`（或您配置的 IP 和端口）

### 4. 停止服务

- 点击相应服务的"停止"按钮
- 或直接关闭应用程序（会提示是否停止所有服务）

### 5. 重启服务

- 点击相应服务的"重启"按钮
- 服务会先停止，然后自动重新启动

## 配置说明

### 配置文件位置

配置文件自动保存在：
```
%APPDATA%\AssetManagement\config.json
```

### 配置生效

- 修改配置后，需要点击"保存配置"才能保存
- **重要**：修改配置后，需要重启相应的服务才能生效
- 前端配置修改后，会自动更新 `vite.config.js` 和 `api.js` 文件

### 环境变量

后端服务通过环境变量读取配置：
- `PORT`：后端端口
- `DB_HOST`：数据库主机
- `DB_PORT`：数据库端口
- `DB_NAME`：数据库名称
- `DB_USER`：数据库用户名
- `DB_PASSWORD`：数据库密码

## 常见问题

### 1. 无法找到项目根目录

**问题**：启动时提示"无法找到项目根目录"

**解决**：
- 确保应用程序位于包含 `backend` 和 `frontend` 目录的项目根目录下
- 或者将应用程序放在项目根目录的任意子目录中（程序会自动向上查找）

### 2. 服务启动失败

**问题**：点击启动按钮后，服务无法启动

**可能原因**：
- Node.js 未安装或未添加到 PATH
- 项目依赖未安装（需要在 `backend` 和 `frontend` 目录运行 `npm install`）
- 端口被占用

**解决**：
- 检查 Node.js 是否已安装：`node --version`
- 检查依赖是否已安装：查看 `backend/node_modules` 和 `frontend/node_modules` 目录是否存在
- 检查端口是否被占用：使用 `netstat -ano | findstr :4001` 查看端口占用情况

### 3. 数据库连接失败

**问题**：后端启动后无法连接数据库

**可能原因**：
- 数据库配置错误（IP、端口、用户名、密码）
- 数据库服务器未启动
- 网络连接问题
- 防火墙阻止连接

**解决**：
- 检查数据库配置是否正确
- 使用数据库客户端工具测试连接
- 检查防火墙设置

### 4. 前端无法访问后端 API

**问题**：前端启动后，页面无法加载数据

**可能原因**：
- API 地址配置错误
- 后端服务未启动
- CORS 配置问题

**解决**：
- 检查"前端配置"中的"API 地址"是否正确
- 确保后端服务已启动
- 检查后端 CORS 配置

## 技术说明

### 项目结构

```
ServiceManager/
├── ServiceManager.csproj    # 项目文件
├── Program.cs                # 程序入口
├── MainForm.cs              # 主窗口
├── ServiceController.cs     # 服务控制逻辑
├── AppConfig.cs             # 配置管理
└── README.md                # 说明文档
```

### 工作原理

1. **配置管理**：使用 JSON 文件保存配置，位置在 `%APPDATA%\AssetManagement\config.json`
2. **服务启动**：通过 `Process.Start` 启动 Node.js 进程，并设置环境变量
3. **配置更新**：启动前端前，自动更新 `vite.config.js` 和 `api.js` 文件
4. **状态监控**：使用 Timer 定时检查进程状态

## 注意事项

1. **关闭应用程序**：关闭应用程序时会自动停止所有运行中的服务
2. **配置文件备份**：建议定期备份配置文件
3. **端口冲突**：确保配置的端口未被其他程序占用
4. **权限问题**：某些操作可能需要管理员权限
5. **路径问题**：确保项目路径中不包含特殊字符
