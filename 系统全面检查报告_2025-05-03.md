## 二、业务逻辑正确性检查

### 2.1 资产编码体系

**✅ 正确实现**
- 使用 `asset_code` 作为主编号（符合项目规则）
- 资产编码唯一性约束（UNIQUE KEY）
- 租户级别隔离（tenant_id + asset_code）
- 导入时支持大小写不敏感检查

**⚠️ 潜在问题**
1. **部门字段混乱**：存在 `department` 和 `department_new` 两个字段，部分代码使用 `department`，部分使用 `department_new`，可能导致数据不一致
2. **外键引用问题**：`inventory_details` 表的 `asset_id` 字段实际存储的是 `asset_code`（字符串），但外键约束指向 `assets(id)`（整数），存在类型不匹配

### 2.2 折旧计算逻辑

**✅ 正确实现**
- 支持 4 种折旧方法：
  - 直线法（straight-line）
  - 双倍余额递减法（double-declining）
  - 年数总和法（sum-of-years-digits）
  - 不计提折旧（none）
- 残值率默认 5%（符合财务规范）
- 默认折旧年限 5 年
- 折旧上限保护：累计折旧 ≤ 可折旧金额
- 账面价值 ≥ 残值保护

**✅ 财务规范符合性**
- 直线法公式：`(原值 - 残值) / 使用寿命` ✅
- 双倍余额递减法：`期初账面价值 × 2 / 使用寿命` ✅
- 年数总和法：`(原值 - 残值) × 剩余年限 / 年数总和` ✅
- 月度计算精度：按整月计算 ✅

### 2.3 资产状态管理

**✅ 正确实现**
- 状态枚举：`在用`, `闲置`, `维修`, `报废`, `调配中`
- 状态常量统一定义（`asset-status.constants.js`）
- 状态别名映射（兼容历史数据）
- 删除保护：在用状态资产不能删除

**⚠️ 潜在问题**
1. 维修状态在数据库中存为 `维修`（简称），但部分业务逻辑可能使用 `维修中`，存在不一致风险
2. 报废流程完成后，未自动更新资产主表状态为 `报废`

### 2.4 资产分类业务逻辑

**✅ 正确实现**
- 支持多级分类树形结构
- 层级自动计算
- 父分类存在性验证
- 分类编码唯一性检查
- 删除保护：有关联资产或子分类不能删除

**⚠️ 潜在问题**
1. 导入时要求"必须选择一级分类"（parent_id != 0），但创建分类时未限制层级深度
2. 公共分类（is_public = 1）与租户分类混合查询，可能导致权限泄露

## 四、数据一致性检查

### 4.1 租户隔离

**✅ 正确实现**
- 所有查询均包含 `tenant_id` 过滤
- 超级管理员支持跨租户查询
- 资产管理员按管理科室隔离

**⚠️ 潜在问题**
1. 部分统计查询（如 `asset.statistics.js`）未过滤 `is_deleted = 0`，可能包含已删除资产
2. 分类查询混合了公共分类和租户分类，可能导致数据泄露

### 4.2 关联数据一致性

**✅ 正确实现**
- 资产调配审批使用事务确保原子性
- 报废流程使用事务处理多表操作
- 软删除保护：删除前检查关联数据

**⚠️ 严重问题**
1. **inventory_details.asset_id 类型不匹配**：
   - 外键定义：`FOREIGN KEY (asset_id) REFERENCES assets(id)`
   - 实际存储：`asset_code`（字符串）
   - **影响**：可能导致外键约束失效或数据不一致

2. **transfer_records.asset_id 类型不匹配**：
   - 同上问题

3. **idle_assets.asset_id 类型不匹配**：
   - 同上问题

### 4.3 审计日志

**✅ 正确实现**
- 资产变更日志（asset_change_logs）
- 审计日志（audit_logs）
- 操作人、操作时间记录完整

## 五、性能问题分析

### 5.1 查询性能

| 问题 | 严重程度 | 说明 |
|------|---------|------|
| **LIKE 前导通配符** | 🟡 中 | 部分查询使用 `%keyword%`，无法使用索引 |
| **全量查询无限制** | 🟡 中 | `/all` 接口可能返回大量数据 |
| **N+1 查询问题** | 🟢 低 | 列表查询使用 JOIN 优化 |
| **缓存策略** | ✅ 良好 | 列表缓存 5 分钟，详情缓存 10 分钟 |

### 5.2 索引优化建议

```sql
-- 建议添加的复合索引
ALTER TABLE assets ADD INDEX idx_tenant_status (tenant_id, status);
ALTER TABLE assets ADD INDEX idx_tenant_department (tenant_id, department_new);
ALTER TABLE assets ADD INDEX idx_tenant_category (tenant_id, category_id);
ALTER TABLE assets ADD INDEX idx_tenant_deleted (tenant_id, is_deleted);
```

### 5.3 分页性能

**✅ 已优化**
- 默认分页大小 20
- 最大分页大小 2000
- 支持游标分页（`/all` 接口使用 OFFSET 分批）

**⚠️ 潜在问题**
- 大偏移量分页（OFFSET > 10000）性能下降
