# 开发环境配置指南

## 📋 项目概览

本项目是一个资产管理系统，采用前后端分离架构：
- **前端**：Vite + React 19 + Ant Design 6
- **后端**：Node.js + Express + MySQL
- **开发模式**：前端端口5177，后端端口5178

## 🔧 核心配置

### 1. 前端配置 (frontend/vite.config.js)

```javascript
import { defineConfig, loadEnv } from 'vite'
import react from '@vitejs/plugin-react'

// 加载环境变量
export default defineConfig(({ mode }) => {
  const env = loadEnv(mode, process.cwd(), '')
  
  // 端口配置
  const frontendPort = parseInt(env.VITE_FRONTEND_PORT) || (mode === 'development' ? 5177 : 4000);
  const backendUrl = env.VITE_BACKEND_URL || (mode === 'development' ? 'http://localhost:5178' : 'http://localhost:5178');
  
  return {
    plugins: [react()],
    server: {
      host: '0.0.0.0',
      port: frontendPort,
      strictPort: true,
      proxy: {
        '/api': {
          target: backendUrl,
          changeOrigin: true,
          secure: false,
          ws: true
        },
        '/uploads': {
          target: backendUrl,
          changeOrigin: true,
          secure: false
        }
      }
    }
  }
})
```

### 2. 后端配置 (backend/config/app.config.js)

## 🔍 开发流程优化

### 1. 启动脚本

创建一个统一的启动脚本 `start-dev.sh`：

```bash
#!/bin/bash

# 启动后端服务
cd backend
npm run dev &
BACKEND_PID=$!

# 启动前端服务
cd ../frontend
npm run dev &
FRONTEND_PID=$!

echo "开发服务器已启动："
echo "- 前端：http://localhost:5177"
echo "- 后端 API：http://localhost:5178/api"

echo "\n按 Ctrl+C 停止所有服务"

# 等待用户中断
trap "kill $BACKEND_PID $FRONTEND_PID" INT
wait
```

### 2. 环境变量管理

在 `frontend` 和 `backend` 目录下创建 `.env.example` 文件，包含所有必要的环境变量：

```env
# frontend/.env.example
VITE_FRONTEND_PORT=5177
VITE_BACKEND_URL=http://localhost:5178

# backend/.env.example
PORT=5178
DB_HOST=localhost
DB_PORT=3306
DB_USER=root
DB_PASSWORD=password
DB_NAME=zcgl
JWT_SECRET=your-secret-key
```

## 🔐 安全配置

### 1. 敏感信息保护
- 所有敏感信息（数据库密码、JWT密钥）必须通过环境变量配置
- `.env` 文件必须添加到 `.gitignore`
- 生产环境使用加密的环境变量管理

### 2. CORS 配置

在后端添加 CORS 中间件：

```javascript
const cors = require('cors');

// 开发环境允许所有来源
if (process.env.NODE_ENV === 'development') {
  app.use(cors());
} else {
  // 生产环境配置允许的来源
  app.use(cors({
    origin: ['https://your-domain.com'],
    credentials: true
  }));
}
```
