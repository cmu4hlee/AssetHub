**❌ 避免：**

  message.success('成功');

### 6.2 错误处理

**✅ 推荐：**
```javascript
const result = await createMaintenanceLog(data);

if (result.success) {
  // 成功处理
  message.success('创建成功');
} else if (result.cancelled) {
  // 用户取消
} else {
  // 错误处理
  message.error(result.error);
}
```

### 6.3 性能优化

**✅ 推荐：**
- 批量操作使用 `Promise.all()`
- 使用防抖处理表单提交
- 合理使用缓存机制

**❌ 避免：**
- 循环中频繁调用创建操作
- 不必要的重复请求
- 大数据量同步处理

## 🔍 故障排查

### 7.1 常见问题

#### 问题1: 创建操作返回428

**原因：** 高风险操作需要确认

**解决：**
1. 检查幂等性键是否正确添加
2. 检查确认令牌是否正确使用
3. 确认API客户端拦截器是否生效

**调试：**
```javascript
// 在浏览器控制台查看请求头
headers: {
  'Idempotency-Key': 'web-xxx-xxx-1',
  'X-Risk-Confirm-Token': 'xxx',
}
```

#### 问题2: 重复创建记录

**原因：** 幂等性键未生效

**解决：**
1. 检查幂等性键格式是否正确
2. 确认后端幂等性检查是否启用
3. 检查Redis缓存是否正常

#### 问题3: 确认令牌无效

**原因：** 令牌已过期或已被使用

**解决：**
1. 令牌有效期5分钟
2. 一次性使用
3. 重新获取新令牌
