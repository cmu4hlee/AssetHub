## 二、核心功能

### 2.1 幂等性键管理

**实现位置：** `frontend/src/api/client.js`

**已实现的自动处理：**

```javascript
// 自动添加幂等性键
const ensureIdempotencyKey = (config) => {
  const method = String(config?.method || 'get').toLowerCase();
  if (!['post', 'put', 'patch', 'delete'].includes(method)) {
    return;
  }

  config.headers = config.headers || {};
  if (config.headers['Idempotency-Key']) {
    return;
  }

  // 自动生成幂等性键
  idempotencySequence += 1;
  config.headers['Idempotency-Key'] = `${IDEMPOTENCY_KEY_SEED}-${idempotencySequence}`;
};
```

**幂等性键格式：**
```
web-{timestamp}-{random}
```

### 2.2 高风险操作确认

**实现位置：** `frontend/src/api/client.js`

**自动处理流程：**

```javascript
// 响应拦截器自动处理高风险确认
api.interceptors.response.use(
  response => response.data,
  async error => {
    const payload = error?.response?.data || {};
    const originalRequest = error?.config;

    // 检查是否需要高风险确认
    if (
      originalRequest &&
      !originalRequest.__riskConfirmed &&
      payload?.requiresConfirmation &&
      payload?.confirmToken
    ) {
      originalRequest.__riskConfirmed = true;
      originalRequest.headers = originalRequest.headers || {};

      // 添加确认令牌
      originalRequest.headers['X-Risk-Confirm-Token'] = payload.confirmToken;

      // 自动重试
      return api.request(originalRequest);
    }

    // 其他错误处理...
  }
);
```

## 五、技术细节

### 5.1 幂等性键生成

```javascript
const IDEMPOTENCY_KEY_SEED = `web-${Date.now()}-${Math.random().toString(36).slice(2, 10)}`;
let idempotencySequence = 0;

// 生成幂等性键
config.headers['Idempotency-Key'] = `${IDEMPOTENCY_KEY_SEED}-${idempotencySequence}`;
```

**格式：** `web-{时间戳}-{随机字符串}-{序号}`

**示例：** `web-1777612000000-abc123-1`

### 5.2 高风险确认流程

1. **第一次请求**
   - 发送带幂等性键的请求
   - 后端返回428状态码
   - 包含确认令牌

2. **第二次请求**
   - 添加 `X-Risk-Confirm-Token` 头
   - 重试原始请求
   - 执行成功操作

3. **确认令牌**
   - 有效期：5分钟
   - 一次性使用
   - 包含操作信息

### 5.3 错误处理

**HTTP状态码处理：**

- **200** - 成功
- **400** - 参数错误
- **401** - 未认证
- **403** - 无权限
- **404** - 资源不存在
- **428** - 需要高风险确认（自动处理）
- **500** - 服务器错误

### 5.4 性能考虑

1. **幂等性键缓存**
   - 每个请求自动生成唯一键
   - 不会重复创建

2. **确认令牌**
   - 一次性使用
   - 5分钟过期

3. **自动重试**
   - 最多重试3次
   - 指数退避

### 7.2 错误处理

✅ **推荐做法：**

```javascript
const { createLog } = useHighRiskOperation({
  onError: (error, result) => {
    if (result.status === 400) {
      message.error('参数错误: ' + error.message);
    } else if (result.status === 403) {
      message.error('没有权限执行此操作');
    } else {
      message.error('操作失败');
    }
  },
});
```

### 7.3 批量操作

✅ **推荐做法：**

```javascript
const handleBatchDelete = async (ids) => {
  Modal.confirm({
    title: `确认删除 ${ids.length} 条记录`,
    onOk: async () => {
      let successCount = 0;
      for (const id of ids) {
        const result = await execute({
          method: 'DELETE',
          url: `/maintenance/logs/${id}`,
          showConfirm: false,
        });
        if (result.success) successCount++;
      }
      message.success(`成功删除 ${successCount} 条`);
    },
  });
};
```

❌ **避免：**

```javascript

for (const id of ids) {
  await deleteMaintenanceLog(id); // 会弹出多个确认框
}
```

## 九、故障排查

### 9.1 常见问题

#### 问题1: 总是返回428

**原因：** 幂等性键格式不正确

**解决：** 检查API客户端是否正确添加了幂等性键

```javascript
// 在Network面板检查请求头
headers: {
  'Idempotency-Key': 'web-xxx-xxx-1',
}
```

#### 问题2: 确认令牌无效

**原因：** 令牌已过期或已被使用

**解决：** 重新发起新请求，获取新的确认令牌

#### 问题3: Hook状态不更新

**原因：** 没有正确使用async/await

**解决：** 确保在async函数中调用

```javascript
// 错误
handleClick() {
  createLog(data); // 没有等待
}

// 正确
handleClick = async () => {
  await createLog(data);
};
```
