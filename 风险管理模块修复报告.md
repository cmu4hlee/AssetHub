## 一、发现的问题

### 1.1 风险评估路由 (`risk-assessment.routes.js`)

| 问题 | 影响 | 严重程度 |
|------|------|----------|
| 硬编码表名 `risk_assessments` | 无法适配不同数据库环境的表名变化 | 中 |
| 不支持 `asset_code` 字段 | 前端无法使用资产编码创建评估 | 高 |
| JOIN 缺少关联字段 | 资产信息关联可能失败 | 中 |
| 缺少评估编号生成 | 前端表格显示不完整 | 中 |

### 1.2 风险分级路由 (`risk-classification.routes.js`)

| 问题 | 影响 | 严重程度 |
|------|------|----------|
| 不支持 `asset_code` 字段 | 无法通过资产编码创建分级 | 中 |
| 动态列名映射不完整 | 部分字段可能为 null | 中 |

## 三、修复后的 API

### 3.1 风险评估 API

| 方法 | 路径 | 参数 | 说明 |
|------|------|------|------|
| GET | `/api/risk/assessments` | `page, pageSize, risk_level` | 获取列表，支持分页过滤 |
| POST | `/api/risk/assessments` | `asset_id` 或 `asset_code`, `risk_level`, `risk_score`, ... | 创建评估 |
| PUT | `/api/risk/assessments/:id` | 同 POST | 更新评估 |
| DELETE | `/api/risk/assessments/:id` | - | 删除评估 |

### 3.2 风险分级 API

| 方法 | 路径 | 参数 | 说明 |
|------|------|------|------|
| GET | `/api/risk/classification` | `page, pageSize, risk_level, keyword` | 获取列表 |
| GET | `/api/risk/classification/stats` | - | 获取统计 |
| POST | `/api/risk/classification` | `asset_id` 或 `asset_code`, `risk_level`, ... | 创建分级 |
| PUT | `/api/risk/classification/:id` | 同 POST | 更新分级 |
| DELETE | `/api/risk/classification/:id` | - | 删除分级 |
