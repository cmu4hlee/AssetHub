## 二、发现并修复的问题

### 2.1 测试配置问题

**问题**: Jest 无法解析 ES Module 模块 (`uuid`, `exceljs`)

**错误信息**:
```
SyntaxError: Unexpected token 'export'
/Users/cjlee/PJ/AssetHub/backend/node_modules/uuid/dist-node/index.js:1
export { default as MAX } from './max.js';
```

**修复方案**:

1. **替换 uuid 模块**: 使用 Node.js 内置的 `crypto.randomUUID()` 替代 `uuid` 包
   - 修改文件: `backend/middleware/auditLogger.js`
   - 修改内容:
   ```javascript
   // 原代码
   const { v4: uuidv4 } = require('uuid');
   const requestId = uuidv4();
   
   // 新代码
   const crypto = require('crypto');
   const generateUUID = () => crypto.randomUUID();
   const requestId = generateUUID();
   ```

2. **更新 Jest 配置**: 配置 `transformIgnorePatterns` 忽略 ESM 模块
   - 修改文件: `backend/jest.config.js`
   - 修改内容:
   ```javascript
   transformIgnorePatterns: [
     '/node_modules/(?!(uuid|exceljs|xlsx)/)',
   ],
   ```

### 2.2 测试代码问题

**修复方案**:
- `db.execute` 返回格式应为 `[{ insertId: 1 }]` 而不是 `[[{ insertId: 1 }]]`
- 修复文件: `backend/tests/unit/audit-logger.service.test.js`

**问题**: intelligent-alert 测试中的期望不符合实际逻辑
