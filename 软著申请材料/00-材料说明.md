## 二、材料使用说明

### 2.1 源代码材料（01-源代码.html）

- 已从项目中提取核心源代码，覆盖 `frontend/src` 和 `backend` 目录。
- 已排除 `node_modules`、`test`、`tests`、`docs`、`scripts`、`.claude` 等非核心目录。
- 已按软件著作权登记要求整理为 **前后各 30 页，每页 50 行，共 60 页**。
- 页眉包含软件名称与版本号，页脚包含页码。
- 已对代码中的敏感信息（如密码、密钥等）做了简单脱敏处理，提交前建议再次人工检查。

**使用方法**：

1. 使用 Chrome、Edge 等浏览器打开 `01-源代码.html`。
2. 按 `Ctrl + P`（Windows）或 `Cmd + P`（macOS）打开打印对话框。
3. 选择“另存为 PDF”或连接打印机打印。
4. 纸张选择 A4，边距选择“默认”。
5. 保存为 PDF 后，检查页数是否为 60 页，每页是否包含 50 行代码。
6. 打印纸质版并装订，准备提交。

### 2.2 软件说明书（02-软件说明书.md）

- 包含软件概述、运行环境、功能模块、系统架构、安装部署、使用说明、特点与创新点等内容。
- 采用 Markdown 格式，方便转换为 Word 或 PDF。

**使用方法**：

1. 使用 VS Code、Typora、MarkText 等工具打开 `02-软件说明书.md`。
2. 将括号中的待填写内容（如著作权人名称、日期等）补充完整。
3. 导出为 Word（.docx）或 PDF 格式。
4. 在封面或末尾加盖公章（单位申请）或签字（个人申请）。

### 2.3 登记申请表信息（03-登记申请表信息.md）

- 汇总了在中国版权保护中心在线填写申请表时需要的核心信息。
- 包含软件基本信息、著作权人信息、权利范围、技术信息、用途与特点、材料清单、办理方式建议、重要提醒、待确认事项。

**使用方法**：

1. 阅读并确认其中的每一项信息。
2. 补充待填写内容（如开发完成日期、著作权人信息等）。
3. 登录中国版权保护中心官网在线填写正式申请表。

## 四、后续步骤建议

1. **确认软件名称与版本**：是否与内部产品命名一致，是否需要加上“软件”字样。
2. **补充著作权人与日期信息**：在 `02-软件说明书.md` 和 `03-登记申请表信息.md` 中填写完整。
3. **检查源代码**：打开 `01-源代码.html`，确认没有敏感信息、第三方版权代码或不相关内容。
4. **转换格式**：将 HTML 源代码和 Markdown 说明书转换为 PDF 或 Word。
5. **盖章/签字**：在纸质版申请表和说明书上加盖公章或签字。
6. **在线提交或邮寄**：登录中国版权保护中心完成在线申请，并按要求递交纸质材料。
7. **保留备份**：妥善保存电子版和纸质版材料。

## 五、注意事项

- 所有材料中的 **软件名称、版本号、著作权人名称** 应保持一致。
- 源代码材料若总代码量不足 3000 行，应提交全部代码。本项目代码量充足，已按要求提取 60 页。

- 本材料仅供申请软件著作权使用，请勿用于其他用途。
