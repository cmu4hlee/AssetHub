# AssetHost 智能资产管理平台 V1.0 软件说明书

## 一、软件概况

AssetHost 智能资产管理平台是由本单位自主研发的一套综合性资产管理系统软件，主要面向医疗机构、制造业、园区管理等中大型组织，提供资产全生命周期的数字化管理能力。

软件版本为 V1.0，采用前后端分离架构开发，前端使用 React 框架，后端基于 Node.js 技术栈，数据库采用 MySQL。

### 主要功能特点

1. 资产全流程管理：覆盖资产从采购入库到报废处置的完整生命周期
2. 维修保养管理：实现故障报修、维修工单、预防性维护等业务闭环
3. 招标采购管理：支持从采购申请到合同签订的全流程招标采购
4. 质量合规管理：满足计量管理、质控管理、特检管理等医疗行业合规要求
5. 风险预警管理：提供设备风险评估与预警能力
6. IoT 物联网接入：支持设备运行数据的实时采集与监测
7. AI 智能助手：提供智能问答与业务辅助功能
8. 多租户架构：支持多组织、多部门的分级管理

### 目标用户

本软件主要服务于以下类型的用户：

- 医疗机构的医学装备管理部门，负责医疗设备的日常管理与维护
- 制造企业的设备管理部门，负责生产设备的资产管理
- 科研院所的实验室管理部门，负责科研仪器的管理
- 园区和公共机构的固定资产管理部门

## 二、系统架构与技术方案

### 2.1 系统架构

系统采用经典的多层架构设计，自下而上分为数据层、业务层、服务层和展示层：

- 数据层使用 MySQL 存储业务数据，Redis 作为缓存
- 业务层封装核心业务逻辑，以 RESTful API 形式对外提供服务
- 前端采用 React + Ant Design 构建用户界面

### 2.2 技术选型

| 技术组件 | 采用方案 |
|----------|----------|
| 前端框架 | React 19 |
| 构建工具 | Vite 6 |
| UI 组件库 | Ant Design 6 |
| 后端框架 | Express 4 |
| 数据库 | MySQL 8.0 |
| 缓存 | Redis 6.x |
| 编程语言 | JavaScript / TypeScript |

### 2.3 部署要求

服务器推荐配置：CPU 4 核以上，内存 8GB 以上，硬盘 100GB 以上。客户端支持 Chrome、Edge、Firefox 等现代浏览器。

## 四、系统特色与创新

1. **完整的业务闭环**：从资产入库到报废，每个环节都有相应的功能支撑，形成闭环管理。

2. **模块化设计**：各功能模块相对独立，可以根据用户需求灵活配置，满足不同行业的差异化需求。

3. **移动端支持**：支持在移动设备上进行扫码盘点、故障报修等操作，提升工作效率。

4. **AI 智能化**：集成 AI 助手，提供智能问答和业务辅助，降低用户学习成本。

5. **合规性保障**：针对医疗行业的特殊要求，提供了计量管理、质控管理、不良事件、特种设备、安全巡检等专业化功能。

6. **物联网集成**：支持接入 IoT 设备，实现设备运行状态的实时监测和预警。

7. **多租户架构**：支持多组织、多部门的分级管理，数据相互隔离，便于集团化部署。

8. **开机率监测**：通过IoT设备采集开关机事件，自动计算设备开机率，为设备利用率分析提供数据支持。
