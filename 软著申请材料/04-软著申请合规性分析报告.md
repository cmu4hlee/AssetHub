## 三、发现的问题与风险

### ⚠️ 问题 1：测试目录中的测试密码

**问题描述**：

在 `backend/test`、`backend/tests` 等目录下存在测试文件，包含测试账号密码：

```javascript
// 示例
password: 'Test123!'
username: 'system_admin'
```

**影响**：

- 软著审查中，代码不应包含明显的测试账号密码
- 虽然这些目录已被排除，但需确认提取的源代码中完全没有此类内容

**建议**：

- ✅ 已排除 test、tests 目录，**风险已消除**
- 提交前再次检查源代码 HTML/MD 文件确认

---

### ⚠️ 问题 2：字段名称可能引起的误解

**问题描述**：

代码中存在以下业务字段名称：

- `real_name`（真实姓名）
- `id_card`（身份证号）
- `bank_card`（银行卡号）

**影响**：

- 仅作为数据库字段名称，是正常的业务需求
- 不会引起版权问题，但可能被误认为包含个人信息

**建议**：

- ✅ 仅为字段定义，无真实数据，**风险较低**
- 建议在说明书中注明这些仅为业务字段定义

---

### ⚠️ 问题 3：源代码提取脚本的局限性

**问题描述**：

当前提取脚本的脱敏规则可能不够全面：

```javascript
// 当前规则
const sensitivePatterns = [
  { regex: /(password|passwd|pwd|secret|token|api[_-]?key|private[_-]?key)\s*[:=]\s*['"`][^'"`]+['"`]/gi, replace: '$1: "***"' },
  { regex: /(DB_PASSWORD|DATABASE_URL|JWT_SECRET|REDIS_PASSWORD)\s*=\s*.+/gi, replace: '$1=***' }
];
```

**可能遗漏**：

- 变量名不规范的密码引用（如 `const pwd = 'xxx'`）
- 注释中的密码
- JSON 配置中的密码

**建议**：

- ✅ 源代码已排除敏感目录
- ⚠️ 建议人工抽查确认

## 六、附录：增强版脱敏脚本（如需更新源代码）

如需重新生成更安全的源代码材料，可使用以下增强版脱敏规则：

```javascript
const sensitivePatterns = [
  // 标准密码字段
  { regex: /(password|passwd|pwd|secret|token|api[_-]?key|private[_-]?key)\s*[:=]\s*['"`][^'"`]{4,}['"`]/gi, replace: '$1: "***"' },
  // 环境变量密码
  { regex: /(DB_PASSWORD|DATABASE_PASSWORD|JWT_SECRET|REDIS_PASSWORD|SMS_SECRET|ALIYUN_SECRET)\s*=\s*.+/gi, replace: '$1=***' },
  // 连接字符串中的密码
  { regex: /(mysql|postgres|mongodb):\/\/[^:]+:[^@]+@/gi, replace: 'mysql://****:****@' },
  // Base64 编码的密钥（简单检测）
  { regex: /(key|token|secret)[_-]?(base64)?\s*[:=]\s*['"][A-Za-z0-9+/=]{20,}['"]/gi, replace: '$1: "***"' },
  // AWS/阿里云密钥格式
  { regex: /(LTAI|AKIA|ASIA)[A-Za-z0-9]{16,}/gi, replace: '****' },
];
```
