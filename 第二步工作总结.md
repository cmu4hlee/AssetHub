## 🔧 技术实现

### 1. 幂等性机制

**原理：**
```
客户端请求 → 添加Idempotency-Key → 后端验证 → 执行操作
                ↓
            如果需要确认 → 返回428 + confirmToken
                ↓
            添加confirmToken → 重试请求 → 执行成功
```

**幂等性键格式：**
```
web-{timestamp}-{random}-{sequence}
示例：web-1777612000000-abc123-1
```

### 2. 高风险确认

**流程：**
1. 用户发起创建/更新/删除请求
2. 后端检查是否需要高风险确认
3. 如果需要，返回428状态码和确认令牌
4. 前端axios拦截器自动处理
5. 添加确认令牌，重试请求
6. 后端验证通过，执行操作

**确认令牌：**
- 有效期：5分钟
- 一次性使用
- 包含操作信息

### 3. 错误处理

**统一错误处理：**
```javascript
{
  success: boolean,
  data?: any,
  message?: string,
  error?: string,
  cancelled?: boolean
}
```

**HTTP状态码映射：**
- 200 → success: true
- 400 → 参数错误
- 401 → 未认证
- 403 → 无权限
- 404 → 资源不存在
- 428 → 高风险确认（自动处理）
- 500 → 服务器错误

### React Hook用法

```jsx
import { useHighRiskOperation } from '../hooks/useHighRiskOperation';

function MyComponent() {
  const { createLog, loading, error } = useHighRiskOperation({
    onSuccess: () => message.success('成功'),
    onError: (err) => message.error(err.message),
  });

  const handleCreate = async () => {
    await createLog({ asset_code: '000000555' });
  };

  return <Button onClick={handleCreate} loading={loading}>创建</Button>;
}
```
