# 资产助手：如何查看日志与判断是否查库/注入

## 一、看后端日志（控制台）

日志通过 `backend/config/logger.js` 输出到 **运行后端的终端**（标准输出）。

1. 在项目里启动后端，例如：
   ```bash
   cd backend
   node server.js
   # 或
   npm run dev
   ```
2. **不要关掉这个终端**，在浏览器里打开资产助手页面，输入资产编号并发送消息。
3. 在该终端里会看到以 `📋` 开头的 JSON 行，其中包含：
   - `资产助手：本轮未提取到资产编号，未执行查库` → **没查库**（未识别到编号）
   - `资产助手：提取到资产编号，将查库并注入上下文` → 识别到编号，准备查库
   - `根据资产编号查询到资产` / `根据资产编号未查询到资产` → **查库结果**
   - `资产助手：已注入资产信息到上下文` → **已注入**资产信息
   - `资产助手：已注入「未找到资产」提示到上下文` → 已注入「未找到」提示

若后端用 PM2 等后台运行，可用：
```bash
pm2 logs
# 或
pm2 logs 0
```
查看对应进程的日志。

## 二、看 API 响应里的 assetLookup（不依赖终端）

发送消息后，接口返回里会多一个 `assetLookup` 字段，可直接判断是否查库、是否注入。

1. 打开浏览器 **开发者工具 (F12)** → **Network（网络）**。
2. 在资产助手里发一条带资产编号的消息。
3. 在 Network 里找到 `message` 或发送消息的接口请求，点开看 **Response**。
4. 在 JSON 里找到 `assetLookup`，含义如下：

| 字段 | 含义 |
|------|------|
| `extractedAssetCode` | 本轮识别到的资产编号，没有则为 `null` |
| `queried` | 是否执行了数据库查询（true=查了） |
| `found` | 是否查到资产（true=查到） |
| `injected` | 是否已把资产信息或「未找到」提示注入到 AI 上下文（true=已注入） |

**判断方式：**

- `extractedAssetCode` 为 `null` → **没识别到编号，不会查库**。
- `queried === true` 且 `found === false` → **查了库但没查到**（编号不存在或租户不符）。
- `queried === true` 且 `found === true` 且 `injected === true` → **查了库且已注入**，AI 应能回复资产信息。
