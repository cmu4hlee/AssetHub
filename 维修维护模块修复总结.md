## 二、问题分析与修复

### 2.1 高风险操作幂等性机制

#### 问题描述
创建操作返回 428 状态码，提示需要 `Idempotency-Key` 请求头和高风险确认。

#### 问题原因
系统为防止重复提交和保障数据安全，实现了幂等性保护机制：

1. **幂等性键（Idempotency-Key）**
   - 用于标识请求的唯一性
   - 防止同一请求被重复执行
   - 格式：UUID，长度不超过128字符

2. **高风险确认（High-Risk Confirmation）**
   - 用于高风险操作（如创建、修改、删除）
   - 需要二次确认才能执行
   - 返回确认令牌（confirmToken）

#### 解决方案
前端需要实现以下流程：

```javascript
// 1. 生成幂等性键
const idempotencyKey = uuid.v4();

// 2. 第一次请求（带幂等性键）
const response = await fetch('/api/maintenance/logs', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${token}`,
    'Idempotency-Key': idempotencyKey,
    'Content-Type': 'application/json'
  },
  body: JSON.stringify(data)
});

// 3. 如果返回 428（需要高风险确认）
if (response.status === 428) {
  const result = await response.json();
  const confirmToken = result.confirmToken;

  // 4. 使用确认令牌重新请求
  const confirmedResponse = await fetch('/api/maintenance/logs', {
    method: 'POST',
    headers: {
      'Authorization': `Bearer ${token}`,
      'Idempotency-Key': idempotencyKey,
      'Confirmation-Token': confirmToken,
      'Content-Type': 'application/json'
    },
    body: JSON.stringify(data)
  });
}
```

#### 相关代码
- **后端实现：** `backend/middleware/idempotency.js`
- **前端需求：** 统一封装API请求方法，自动处理幂等性

### 2.2 维修模板推荐API修复

#### 问题描述
调用 `/maintenance/templates/recommend` 端点时，提示"资产类型不能为空"。

#### 解决方案
使用新的端点 `/maintenance/templates/recommend-by-asset`，只需提供资产编号即可。

#### API对比

| 端点 | 必需参数 | 优势 |
|-----|---------|------|
| `/recommend` | `asset_type` | 灵活，可跨资产类型推荐 |
| `/recommend-by-asset` | `asset_code` | 简单，直接通过资产编号推荐 |

#### 前端实现

**更新文件：** `frontend/src/api/domains/maintenance.js`

```javascript
// 添加新方法
recommendTemplatesByAsset: assetCode =>
  api.get('/maintenance/templates/recommend-by-asset', {
    params: { asset_code: assetCode }
  }),
```

**使用示例：**
```javascript
// 根据资产编号获取推荐模板
const result = await maintenanceAPI.recommendTemplatesByAsset('000000555');
if (result.success) {
  const templates = result.data;
  console.log('推荐模板:', templates);
}
```

#### 验证结果
✅ 端点正常工作，返回资产信息和推荐模板

3. **高风险操作包装器**
```javascript
// services/highRiskService.js
import { generateIdempotencyKey } from '../utils/idempotency';

export async function executeHighRiskOperation(apiCall) {
  const headers = {
    'Idempotency-Key': generateIdempotencyKey()
  };

  const response = await apiCall(headers);

  if (response.status === 428) {
    const confirmToken = response.data.confirmToken;
    headers['Confirmation-Token'] = confirmToken;
    return await apiCall(headers);
  }

  return response;
}
```
