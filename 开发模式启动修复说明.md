# 开发模式启动异常修复说明

## 🔍 问题诊断

### 开发模式端口配置
- **前端端口**：6000（开发模式）
- **后端端口**：6001（开发模式）
- **生产模式端口**：前端4000，后端4001

## ✅ 解决方案

### 方案1：使用默认端口（推荐）

开发模式已配置为：
- 前端：6000
- 后端：6001

直接启动即可：

```bash
# 启动后端
cd backend
npm start

# 启动前端
cd ../frontend
npm run dev
```

### 方案2：使用环境变量指定端口

如果需要使用其他端口：

```bash
# 前端使用其他端口
cd frontend
VITE_FRONTEND_PORT=6002 npm run dev

# 后端使用其他端口
cd ../backend
PORT=6002 npm start
```

## 🚀 标准启动流程

### 1. 检查端口占用
```bash
# 检查端口占用情况
lsof -i:6000
lsof -i:6001
```

### 2. 启动后端服务
```bash
cd Asset/backend
npm start
```

后端应该运行在：`http://localhost:6001`

### 3. 启动前端服务
```bash
cd Asset/frontend
npm run dev
```

前端会使用6000端口，如果被占用会自动切换到下一个可用端口。

### 4. 查看实际端口

启动后，Vite会在控制台显示实际使用的端口，例如：
```
VITE v7.x.x  ready in xxx ms

➜  Local:   http://localhost:5002/
➜  Network: http://192.168.x.x:5002/
```

## 📝 验证服务

### 检查后端
```bash
curl http://localhost:6001/api/health
```

应该返回：
```json
{"status":"ok","timestamp":"..."}
```

### 检查前端
在浏览器访问控制台显示的地址（通常是 `http://localhost:6000`）

## ⚠️ 注意事项

1. **端口冲突**：确保前端端口与后端端口（6001）不同
2. **环境变量**：如果设置了`VITE_FRONTEND_PORT`或`PORT`，会优先使用该值
3. **自动切换**：Vite的`strictPort: false`配置允许自动切换端口，避免启动失败

## 🔧 快速修复命令

如果遇到端口占用问题，可以快速关闭占用进程：

```bash
# 关闭占用6000端口的进程（谨慎使用）
lsof -ti:6000 | xargs kill -9 2>/dev/null

# 关闭占用6001端口的进程
lsof -ti:6001 | xargs kill -9 2>/dev/null
```
