# 资产管理系统功能文档

## 1. 系统概述

### 1.1 系统简介

资产管理系统是一套专业的资产管理解决方案，适用于医院医疗设备、普通设备、房产建筑、办公家具等各类资产管理。系统提供了完整的资产生命周期管理，包括资产的录入、查询、盘点、调配、维修、报废等全流程管理，帮助企业实现资产的精细化管理和高效利用。

### 1.2 系统特点

- **多租户架构**：支持多个企业/部门独立使用，数据完全隔离
- **完整的资产生命周期管理**：从资产录入到报废的全流程管理
- **强大的查询和统计功能**：支持多条件筛选、排序、分页和导出
- **灵活的权限管理**：基于角色的访问控制，支持细粒度权限设置
- **完善的审计日志**：记录所有关键操作，确保操作可追溯
- **支持多种资产类型**：医疗设备、普通设备、房产建筑、办公家具、车辆等
- **文件管理功能**：支持资产相关技术资料的上传、管理和下载
- **移动端支持**：提供移动端访问能力，支持资产盘点等操作
- **数据安全可靠**：完善的数据备份和恢复机制

## 6. 部署说明

### 6.1 环境准备

| 环境 | 要求 |
|------|------|
| 操作系统 | Linux/Windows/macOS |
| Node.js | 16.x 或更高版本 |
| MySQL | 8.x |
| Redis | 7.x（可选，用于缓存） |
| Git | 用于代码管理 |

### 6.2 安装步骤

1. **克隆代码**
   ```bash
   git clone <repository-url>
   cd Asset
   ```

2. **安装依赖**
   ```bash
   # 后端依赖
   cd backend
   npm install
   
   # 前端依赖
   cd ../frontend
   npm install
   ```

3. **数据库初始化**
   ```bash
   cd ../backend
   # 执行数据库初始化脚本
   node scripts/init-db.js
   ```

4. **配置环境变量**
   ```bash
   # 在backend目录下创建.env文件
   touch .env
   # 编辑.env文件，配置数据库连接、JWT密钥等
   ```

5. **启动服务**
   ```bash
   # 启动后端服务
   cd backend
   npm run dev
   
   # 启动前端服务
   cd ../frontend
   npm run dev
   ```

6. **访问系统**
   - 前端地址：http://localhost:4000
   - 后端地址：http://localhost:4001
   - API文档：http://localhost:4001/api-docs

### 6.3 生产环境部署

1. **构建前端项目**
   ```bash
   cd frontend
   npm run build
   ```

2. **配置Nginx**
   ```nginx
   server {
       listen 80;
       server_name your-domain.com;
       
       # 前端静态文件
       location / {
           root /path/to/frontend/dist;
           index index.html;
           try_files $uri $uri/ /index.html;
       }
       
       # 后端API代理
       location /api {
           proxy_pass http://localhost:4001;
           proxy_http_version 1.1;
           proxy_set_header Upgrade $http_upgrade;
           proxy_set_header Connection 'upgrade';
           proxy_set_header Host $host;
           proxy_cache_bypass $http_upgrade;
           proxy_set_header X-Real-IP $remote_addr;
           proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
       }
   }
   ```

3. **使用PM2管理后端服务**
   ```bash
   cd backend
   npm install -g pm2
   pm2 start server.js --name asset-management
   pm2 startup
   pm2 save
   ```
