## 二、具体改进

### 2.1 前端Hooks增强 (`useApi.js`)

#### 新增功能

**1. 请求重试机制**
```javascript
// 在 useApiRequest 中新增 retryCount 和 maxRetries 选项
const result = await execute(apiCall, {
  maxRetries: 3,  // 最多重试3次
  retryDelay: 1000  // 重试间隔1秒
});
```

**2. 防抖搜索支持**
```javascript
// 在 useListRequest 中新增 debounce 参数
const { fetchList } = useListRequest('/api/assets');
await fetchList(params, { debounce: 300 });  // 300ms防抖
```

**3. 批量操作Hook**
```javascript
// 新增 useBatchOperation Hook
const { loading, progress, executeBatch } = useBatchOperation(async (item) => {
  return await api.post('/api/batch', item);
});

const result = await executeBatch(items, {
  batchSize: 10,
  onItemSuccess: (item, result) => console.log('成功', item),
  onItemError: (item, error) => console.error('失败', item)
});
```

**4. 防抖搜索Hook**
```javascript
// 新增 useDebouncedSearch Hook
const { searchValue, setSearchValue, loading, clearSearch } = useDebouncedSearch(
  async (value) => {
    const result = await api.get('/api/search', { params: { q: value } });
    setResults(result.data);
  },
  300  // 300ms防抖
);
```

### 2.2 后端验证增强 (`error-handler.js`)

#### 新增验证方法

| 方法 | 功能 | 示例 |
|------|------|------|
| `validateDateRange` | 验证日期范围 | 开始日期不能晚于结束日期 |
| `validateLength` | 验证字符串长度 | 名称长度2-50字符 |
| `validateEnum` | 验证枚举值 | 状态必须是active/inactive |
| `validatePhone` | 验证手机号 | 11位手机号格式 |
| `validateEmail` | 验证邮箱 | 标准邮箱格式 |
| `validateUrl` | 验证URL | URL格式校验 |
| `validateId` | 验证ID | 正整数格式 |
| `validateObject` | 批量对象验证 | 一次验证多个字段 |

#### 批量验证示例

```javascript
ValidationUtil.validateObject(data, {
  name: { required: true, type: 'string', minLength: 2, maxLength: 50 },
  age: { type: 'number', min: 0, max: 150 },
  email: { type: 'string', pattern: /^[^\s@]+@[^\s@]+\.[^\s@]+$/ },
  status: { type: 'string', enum: ['active', 'inactive', 'pending'] },
  phone: { type: 'string', pattern: /^1[3-9]\d{9}$/ }
});
```

#### AppError 增强

```javascript
// 新增 details 属性和 toJSON 方法
const error = new AppError('验证失败', 400, 'VALIDATION_ERROR', {
```

### 2.3 前端规范化增强 (`normalizers.js`)

#### 新增函数

| 函数 | 功能 | 示例 |
|------|------|------|
| `normalizeRecord` | 规范化单个记录 | 类型转换、默认值处理 |
| `normalizeError` | 规范化错误响应 | 统一的错误格式 |
| `normalizeDate` | 规范化日期格式 | 多种格式支持 |
| `formatCurrency` | 格式化金额显示 | `¥1,234.56` |
| `formatFileSize` | 格式化文件大小 | `1.5 MB` |
| `formatPercent` | 格式化百分比 | `75.5%` |
| `normalizeEnumLabel` | 规范化枚举显示 | 键值映射 |
| `normalizeRecords` | 批量规范化记录 | 数组处理 |
| `createPaginatedResponse` | 创建分页响应 | 标准化分页格式 |

#### 使用示例

```javascript
// 规范化记录
const normalized = normalizeRecord(record, {
  age: 'number',
  isActive: 'boolean',
  createdAt: 'date'
});

// 格式化金额
formatCurrency(1234.56);  // ¥1,234.56
formatCurrency(1234.56, { currency: '$', showSymbol: false });  // 1,234.56

// 格式化文件大小
formatFileSize(1024 * 1024 * 1.5);  // 1.5 MB

// 批量规范化
const normalizedRecords = normalizeRecords(records, {
  status: 'string',
  count: 'number'
});
```
