### 2.1 前端性能优化

#### ✅ 错误边界组件 (`ErrorBoundary.jsx`)

**功能特性**:
- 捕获子组件渲染错误，防止整个应用崩溃
- 支持开发/生产环境不同的错误展示
- 错误次数统计，超过阈值显示严重错误提示
- 提供重试、刷新、返回首页等恢复选项
- 支持自定义 fallback 组件

**使用方法**:
```jsx
// 页面级错误边界
<PageErrorBoundary>
  <YourPage />
</PageErrorBoundary>

// 组件级错误边界
<ComponentErrorBoundary fallback={CustomFallback}>
  <YourComponent />
</ComponentErrorBoundary>

// HOC 包装
export default withErrorBoundary(YourComponent, options);
```

#### ✅ 图片懒加载组件 (`LazyImage.jsx`)

**功能特性**:
- 基于 IntersectionObserver 的懒加载
- 加载占位符和错误状态处理
- 淡入动画效果
- 支持背景图片懒加载
- 图片画廊组件

**使用方法**:
```jsx
// 基础使用
<LazyImage 
  src="image.jpg" 
  alt="描述" 
  width={300} 
  height={200}
/>

// 背景图片
<LazyBackgroundImage src="bg.jpg">
  <Content />
</LazyBackgroundImage>

// 图片画廊
<LazyImageGallery 
  images={[{ src: '1.jpg', alt: '1' }, { src: '2.jpg', alt: '2' }]}
  columns={3}
/>
```

#### ✅ 性能监控组件 (`PerformanceMonitor.jsx`)

**功能特性**:
- 页面加载时间监控（DNS、TCP、TTFB、DOM、总加载）
- 慢资源加载检测（>1000ms）

### 2.3 API 性能优化

#### ✅ 请求缓存工具 (`request-cache.js`)

**功能特性**:
- 智能缓存键生成
- TTL 过期策略
- 请求去重（避免重复请求）
- 自动缓存清理
- 缓存统计信息

**缓存策略**:
```javascript
CACHE_CONFIG = {
  LONG_TERM: 60 * 60 * 1000,    // 1小时
  MEDIUM_TERM: 15 * 60 * 1000,  // 15分钟
  SHORT_TERM: 5 * 60 * 1000,    // 5分钟
  REAL_TIME: 0,                 // 不缓存
}
```

**预定义策略**:
- 用户列表: 短时间缓存
- 资产分类: 长时间缓存
- 部门列表: 中等时间缓存
- 系统配置: 长时间缓存
- 仪表盘统计: 短时间缓存

**使用方法**:
```javascript
import requestCache, { CACHE_CONFIG } from '@/utils/request-cache';

// 带缓存的请求
const data = await requestCache.get('/api/assets', {
  cache: true,
  cacheTTL: CACHE_CONFIG.MEDIUM_TERM
});

// 清除缓存
requestCache.clearMatching('/api/assets');
```

#### ✅ 请求合并工具 (`request-batcher.js`)

**功能特性**:
- 自动合并相似请求
- 可配置合并窗口期
- 批量操作工具
- 进度回调支持

**使用方法**:
```javascript
import requestBatcher, { BatchOperation } from '@/utils/request-batcher';

// 批量请求
const result = await requestBatcher.batchRequest('POST', '/api/assets/batch-update', data);

// 批量操作
const batch = new BatchOperation({
  batchSize: 50,
  onProgress: ({ percentage }) => console.log(`${percentage}%`),
});

items.forEach(item => batch.add(item));
const { results, errors } = await batch.execute(asyncFn);
```
