## 一、模块架构

资产调配模块采用「单一数据源」设计，统一使用 `asset_transfer_requests` 表作为核心存储。

### 1.1 后端路由
| 路由文件 | 端点 | 功能 |
|---------|------|------|
| `routes/transfer.js` | GET/POST `/api/transfer` | 调配记录列表/创建 |
| `routes/transfer.js` | GET `/api/transfer/:id` | 调配详情 |
| `routes/transfer.js` | PUT `/api/transfer/:id/approve` | 审批调配 |
| `routes/transfer.js` | PUT `/api/transfer/:id/complete` | 完成调配 |
| `routes/transfer.js` | DELETE `/api/transfer/:id` | 删除调配 |
| `routes/assets/asset.transfer.js` | GET `/api/assets/transfer-requests` | 调配申请列表 |
| `routes/assets/asset.transfer.js` | POST `/api/assets/:id/transfer-apply` | 提交调配申请 |
| `routes/assets/asset.transfer.js` | POST `/api/assets/transfer-requests/:id/approve` | 审批调配申请 |

### 1.2 核心服务
- `services/transfer-approval-service.js` - 统一业务逻辑封装
  - `createTransferRequest()` - 创建调配申请（校验资产状态为"在用"，置资产为"调配中"）
  - `approveTransferRequest()` - 审批调配（批准时更新资产部门+状态为"在用"）
  - `rejectTransferRequest()` - 拒绝调配（回滚资产状态为"在用"）
  - `deleteTransferRequest()` - 删除调配（按需回滚资产状态）
  - `findTransferRequestById()` - 查询单条
  - `getAssetForTransfer()` - 获取资产（FOR UPDATE 行锁）
  - `hasOtherPendingTransfer()` - 检查是否有待审批调配

### 1.3 前端页面
| 页面 | 路由 | 功能 |
|------|------|------|
| `TransferList.jsx` | `/transfer` | 调配记录列表 |
| `TransferForm.jsx` | `/transfer/new` | 调配申请表单 |
| `TransferRequestList.jsx` | `/transfer/requests` | 调配申请处理（审批） |

### 1.4 状态机
```
在用 → 调配中(申请时) → 在用(审批通过后)
         ↓
       已拒绝(回滚到在用)
```

## 二、已发现并修复的问题

### 2.1 数据库表字段缺失（本次修复）

**问题**: `asset_transfer_requests` 表缺少 `comment` 和 `updated_at` 字段

**影响**:
- `/api/assets/transfer-requests` 查询报错 `Unknown column 't.comment'`

**修复**:
```sql
ALTER TABLE asset_transfer_requests ADD COLUMN comment TEXT COMMENT '审批意见' AFTER approved_at;
ALTER TABLE asset_transfer_requests ADD COLUMN updated_at DATETIME NULL DEFAULT NULL COMMENT '更新时间' AFTER created_at;
```

### 2.2 历史遗留问题（已在之前修复）

根据 2026-07-12 日志，以下问题已修复：

1. **DELETE 错表** - `routes/transfer.js` 的 DELETE 读取 `asset_transfer_requests` 却删除 `transfer_records`
2. **跨租户泄漏** - 驳回分支判断查 `transfer_records` 缺 `tenant_id`
3. **未置资产为调配中** - `createTransfer` 未将资产状态置为"调配中"
4. **completed 状态错误** - `completeTransfer` 写入不存在的 `completed` ENUM 值
5. **建表脚本错误** - 列名误写 `asset.code`、缺 `tenant_id`
