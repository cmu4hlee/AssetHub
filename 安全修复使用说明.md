# 🚀 资产管理系统安全修复 - 使用说明

## 📋 快速开始

### 1. 验证安全修复
```bash
# 进入后端目录
cd Desktop/Asset/backend

# 运行安全检查（推荐第一步）
node scripts/security-init.js

# 运行完整安全测试（验证修复效果）
node scripts/security-test.js
```

### 2. 重启服务应用修复
```bash
# 重启后端服务
npm restart
# 或
npm run dev

# 验证服务正常
curl http://localhost:5174/api/health
```

## 🔧 配置管理

### 环境变量
主要安全配置在 `.env` 文件中：
```env
# JWT安全
JWT_SECRET=AssetMgmtSys2024SecKey_随机密钥

# 访问控制
ENABLE_IP_WHITELIST=true
ENABLE_DOMAIN_WHITELIST=true

# CORS安全
CORS_ORIGIN=http://localhost:3000,http://localhost:4000
```

### 安全配置
详细安全配置在 `config/security.config.js` 中：
```javascript
module.exports = {
  auth: {
    disableDefaultTestUser: true,
    requireAuthentication: true,
  },
  tenant: {
    enforceTenantIsolation: true,
    validateTenantId: true,
  },
  // ... 更多配置
};
```

## 🆘 故障排除

### 常见问题

#### 1. 认证失败
```
症状: 正常用户无法登录
解决: 检查JWT_SECRET配置和用户权限
```

#### 2. 租户隔离错误
```
症状: 用户无法访问租户数据
解决: 检查用户租户关联和tenant_id设置
```

#### 3. 频率限制误触发
```
症状: 正常请求被限制
解决: 调整rateLimit配置参数
```
