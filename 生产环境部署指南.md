## 二、部署步骤

### 2.1 后端部署

#### 步骤1：备份配置

```bash
# 备份数据库
mysqldump -u root -p zcgl > backup_$(date +%Y%m%d_%H%M%S).sql

# 备份配置文件
cp backend/.env backend/.env.backup
```

#### 步骤2：停止服务

```bash
# 停止后端服务
pm2 stop backend

# 或者使用systemd
sudo systemctl stop asset-backend
```

#### 步骤3：更新代码

```bash
cd /Volumes/移动硬盘（500）/AssetHub/backend

# 拉取最新代码（如果使用git）
git pull origin main

# 或者复制更新的文件
cp -r /path/to/new/files ./services/maintenance/
cp -r /path/to/new/files ./middleware/high-risk-action-gate.js
```

#### 步骤4：安装依赖

```bash
cd /Volumes/移动硬盘（500）/AssetHub/backend
npm install
```

#### 步骤5：启动服务

```bash
# 使用pm2
pm2 start server.js --name backend

# 或者使用systemd
sudo systemctl start asset-backend
```

#### 步骤6：验证服务

```bash
# 检查服务状态
curl http://localhost:5183/api/health

# 检查响应
# {"success":true,"status":"healthy",...}
```

### 2.2 前端部署

#### 步骤1：构建前端

```bash
cd /Volumes/移动硬盘（500）/AssetHub/frontend
npm install
npm run build
```

#### 步骤2：部署静态文件

```bash
# 方式1：使用nginx
cp -r dist/* /var/www/html/

# 方式2：使用pm2
pm2 restart frontend

# 方式3：使用CDN
# 上传到CDN并更新配置
```

#### 步骤3：配置nginx（可选）

```nginx
server {
    listen 80;
    server_name your-domain.com;

    root /var/www/html;
    index index.html;

    location /api {
        proxy_pass http://localhost:5183;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }

    location / {
        try_files $uri $uri/ /index.html;
    }
}
```

## 五、回滚方案

### 5.1 快速回滚

#### 后端回滚

```bash
# 停止当前服务
pm2 stop backend

# 恢复备份
cp backend/.env.backup backend/.env

# 启动旧版本
git checkout HEAD~1
npm install
pm2 restart backend
```

#### 数据库回滚

```bash
# 停止服务
pm2 stop backend

# 回滚数据库
mysql -u root -p zcgl < backup_YYYYMMDD_HHMMSS.sql

# 启动服务
pm2 restart backend
```

### 5.2 灰度回滚

#### 步骤1：切换流量

```nginx
# 将部分流量切换到旧版本
upstream backend_old {
    server localhost:5184;
}

upstream backend_new {
    server localhost:5185;
}

server {
    location /api {
        # 10%流量到新版本
        random_by_addr($remote_addr) < 0.1
            ? proxy_pass http://backend_new
            : proxy_pass http://backend_old;
    }
}
```

#### 步骤2：监控指标

```bash
# 监控错误率
watch -n 5 'curl -s http://localhost:5183/api/health | jq .status'

# 监控响应时间
ab -n 100 -c 10 http://localhost:5183/api/maintenance/logs
```

#### 步骤3：完全回滚

```bash
# 100%流量切回旧版本
# 更新nginx配置后reload
sudo nginx -s reload
```

## 六、安全配置

### 6.1 防火墙配置

```bash
# 只开放必要端口
sudo firewall-cmd --permanent --add-port=80/tcp    # HTTP
sudo firewall-cmd --permanent --add-port=443/tcp   # HTTPS
sudo firewall-cmd --permanent --add-port=5183/tcp  # API（仅内网）
sudo firewall-cmd --reload
```

### 6.2 HTTPS配置

```nginx
# nginx配置HTTPS
server {
    listen 443 ssl http2;
    server_name your-domain.com;

    ssl_certificate /path/to/cert.pem;
    ssl_certificate_key /path/to/key.pem;
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;

    location /api {
        proxy_pass http://localhost:5183;
    }
}
```

### 6.3 CORS配置

```javascript
// backend/config/cors.js
module.exports = {

  allowedHeaders: ['Content-Type', 'Authorization', 'Idempotency-Key']
};
```

## 七、故障排查

### 7.1 常见问题

#### 问题1：API返回500错误

**排查步骤：**
```bash
# 1. 检查服务日志
tail -f backend/logs/error.log

# 2. 检查数据库连接
mysql -u root -p -e "SELECT 1;"

# 3. 重启服务
pm2 restart backend

# 4. 检查内存
free -h
```

#### 问题2：幂等性不生效

**排查步骤：**
```bash
# 1. 检查请求头
curl -v -X POST "http://localhost:5183/api/maintenance/logs" \
  -H "Idempotency-Key: test-key"

# 2. 检查Redis连接（如果使用）
redis-cli ping

# 3. 检查幂等性中间件
grep -r "idempotency" backend/middleware/
```

#### 问题3：创建操作返回428

**原因：** 高风险操作需要确认

**解决方案：**
```bash
# 1. 第一次请求
curl -X POST "http://localhost:5183/api/maintenance/logs" \
  -H "Idempotency-Key: test-key-123"

# 2. 响应返回确认令牌
# {"requiresConfirmation": true, "confirmToken": "xxx"}

# 3. 使用令牌再次请求
curl -X POST "http://localhost:5183/api/maintenance/logs" \
  -H "Idempotency-Key: test-key-123" \
  -H "X-Risk-Confirm-Token: xxx"
```

### 7.2 性能问题

#### 问题：响应时间过长

**排查步骤：**
```bash
# 1. 检查数据库慢查询
mysql -u root -p -e "SHOW VARIABLES LIKE 'slow_query_log';"
mysql -u root -p -e "SHOW GLOBAL STATUS LIKE 'Slow_queries';"

# 2. 检查Redis缓存
redis-cli info stats | grep hit

# 3. 检查连接池
mysql -u root -p -e "SHOW STATUS LIKE 'Threads_connected';"
```
