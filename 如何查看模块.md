## 二、使用模块API

### 合规性管理模块 (`/api/compliance`)

```bash
# 获取保养模板列表
GET /api/compliance/maintenance-level/templates

# 获取保养计划列表
GET /api/compliance/maintenance-level/plans

# 生成保养计划
POST /api/compliance/maintenance-level/plans/generate

# 获取特种设备列表
GET /api/compliance/special-equipment

# 获取安全检测记录
GET /api/compliance/safety-inspection
```

### 开机率管理模块 (`/api/uptime`)

```bash
# 获取运行记录
GET /api/uptime/operation-logs

# 添加运行记录
POST /api/uptime/operation-logs

# 计算开机率
POST /api/uptime/calculate

# 获取开机率统计
GET /api/uptime/statistics

# 获取开机率概览
GET /api/uptime/overview

# 获取仪表盘数据
GET /api/uptime/dashboard
```

### 人员资质模块 (`/api/staff`)

```bash
# 获取资质列表
GET /api/staff/qualifications

# 添加资质
POST /api/staff/qualifications

# 获取即将到期资质
GET /api/staff/qualifications/expiring

# 获取培训记录
GET /api/staff/training-records

# 添加培训记录
POST /api/staff/training-records
```

### 设备风险模块 (`/api/risk`)

```bash
# 获取风险评估列表
GET /api/risk/assessment

# 创建风险评估
POST /api/risk/assessment

# 获取风险分级统计
GET /api/risk/classification/stats

# 获取风险控制列表
GET /api/risk/control
```

打开浏览器访问：http://localhost:13579

- 配置模块参数

## 四、模块配置

### 查看模块配置

```bash
# 查看开机率模块配置项
GET /api/uptime/config

# 返回示例：
{
  "success": true,
  "data": {
    "schema": [
      {
        "key": "life_support_uptime_threshold",
        "name": "生命支持类设备开机率阈值(%)",
        "type": "number",
        "default": 99,
        "min": 90,
        "max": 100
      }
    ],
    "default": {
      "life_support_uptime_threshold": 99,
      "large_equipment_uptime_threshold": 95
    }
  }
}
```

### 修改模块配置

```bash
# 通过数据库修改租户模块配置
mysql -u root -p zcgl -e "
UPDATE tenant_module_configs 
SET config = '{\"life_support_uptime_threshold\": 98}'
WHERE tenant_id = 1 AND module_id = 'uptime-management';
"
```

## 五、故障排查

### 1. 模块未显示

检查步骤：
```bash
# 1. 检查服务是否运行
curl http://localhost:5183/api/health

# 2. 检查模块是否加载
curl http://localhost:5183/api/system/modules | grep -E "(compliance|uptime|staff|risk)"

# 3. 检查后端日志
tail -f logs/backend-start.log
```

### 2. API返回404

可能原因：
- 模块未正确加载
- 路由路径错误
- 功能开关被禁用

检查方法：
```bash
# 检查模块状态
curl http://localhost:5183/api/compliance/status

# 如果返回404，说明模块未加载
```

### 3. 数据库表缺失

执行建表脚本：
```bash
# 1. 模块配置表
mysql -u root -p zcgl < backend/scripts/create-module-config-table.sql

# 2. 合规性管理表
mysql -u root -p zcgl < backend/scripts/create-compliance-tables.sql

# 3. 数据库优化
mysql -u root -p zcgl < backend/scripts/database-optimization.sql
```
